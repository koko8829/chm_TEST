Itâ€™s a client component that can receive messages from the X-PUSH server.
Remark 
- You can receive messages provided from the X-PUSH server by establishing a connection with the X-PUSH server and passing authentication, and then registering the topic to be received using the message type and message key.

- The message type is the type or group of message, and the message key functions as an index to separate each message.

- The version of the X-PUSH server should be v2.6 or higher to use the XPush function in the WRE.
- When executing in the mobile NRE or WRE, the app (or browser) does not process actions such as connect, disconnect, subscribe, etc. automatically when it goes down to the background or goes up to the foreground, and the onsucess or onerror event does not occur.
Property
Name Description
controlretry This property sets the number of retries conducted for the error that occurs when a topic is registered in or deleted from the X-PUSH server.
debug This property is used to determine whether to show XPush messages on the Debug pane of Nexacro Studio.
errorcode This read-only property holds a code indicating the error that occurred.
errormsg This read-only property holds the message of an error that has occurred.
iplist This property is used to set the IP addresses and ports of X-PUSH servers to connect.
keepalivetime This property sets the intervals between messages sent for maintaining the connection with an X-PUSH server.
keeptimeout This property sets a period of time allowed for receiving the X-PUSH server's response to a message sent for maintaining the connection with the server.
layouturl The attribute to set the file location defining the message layout.
name This property is used to specify the name of a XPush.
projectid This property is used to set a project ID used in the communication with the X-PUSH server.
protocolversion This property is used to set the version of the protocol used in the communication with the X-PUSH server.
retry This property sets the number of retries conducted in the event of the X-PUSH server connection error.
sessionid This property is used to set a session ID that will be used for connecting to an X-PUSH server.
timeout This property sets a period of time allowed for receiving the X-PUSH server's response to a message sent for connecting with the server.
userid This property is used to set a user ID that will be used for connecting to an X-PUSH server.
Method
Name Description
connect This method is used to connect to an X-PUSH server.
disconnect This method is used to disconnect from an X-PUSH server.
getCurrentIP This method returns the IP value of a currently-connected X-PUSH server.
getCurrentPort This method returns the port number of a currently-connected X-PUSH server.
getRegisteredTopics This method is used to request the topic information in the connected X-PUSH server. The topic here was registered for reliable messaging.
getRegisteredTopicsWithUserID This method is used to request the topic information in the connected X-PUSH server. The topic here was registered for reliable messaging by a specific user.
registerDevice This method registers a device token in an X-PUSH server to receive notifications through the mobile device.
registerTopic This method is used to register a topic in the connected X-PUSH server for reliable messaging.
registerTopicWithUserID This method registers a topic in an X-PUSH server so that a certain user can receive reliable messages.
requestMessage This method is used to request reliable messaging messages that are not received yet from an X-PUSH server.
requestMessageCount This method requests the number of reliable messaging messages that are not received yet from an X-PUSH server.
requestMessageCountWithUserID This method is used to request the number of reliable messages that a certain user has not received. The request is sent to an X-PUSH server.
sendResponse This method is used to respond to a reliable messaging message sent by an X-PUSH server.
subscribe The method to register a topic on the X-PUSH server connected to receive messages.
unregisterDevice This method delete a device token from an X-PUSH server. The device token was registered to receive notifications through the mobile device.
unregisterTopic This method is used to delete a topic in the connected X-PUSH server. The topic here was registered for reliable messaging.
unregisterTopicWithUserID This method is used to remove a topic for reliable messages that are received by a certain user. The topic is removed from an X-PUSH server.
unsubscribe This method is used to cancel the topic for receiving messages.
Event
Name Description
onerror This event occurs when a task requested to an XPush server fails.
onkeepalive This event occurs when an X-PUSH server receives a message sent for maintaining the connection.
onsuccess This event occurs when a task requested by an XPush succeeds.
