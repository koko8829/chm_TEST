Description 
This object deals with basic information of an application that is based on Nexacro. This object also addresses the environmental information that is provided when the application is created.
Property
Name Description
all This is a read-only property that holds all the objects registered in a certain application.
licenseurl This property is used to specify the path to the file that contains the information on a license.
mainframe This read-only property holds the MainFrame of an application.
name This property is used to specify the name of an application.
screenid This property is used to set the IDs of screens applied to an application.
trays This read-only property holds the information on the trays registered in an application.
version This property is used to set the version of an application.
Method
Name Description
addEventHandler This method is used to add a handler to a certain event of an application.
addEventHandlerLookup This method is used to find a function and add it to a certain event of an application as a handler.
addTray This method is used to add a dynamically-created tray to an application.
addVariable This method is used to add a variable to the AppVariables field of an application.
alert This method is used to display an alert box in an application.
cancelTransaction This method is used to cancel an ongoing transaction.
clearEventHandler This method is used to delete all the event handlers registered in a certain event.
confirm The method to display the selected dialogue box of the application.
exit This method is used to close the running Nexacro application.
findEventHandler This method finds a handler in a certain event of an application and returns the index of the handler.
getActiveForm This method returns the currently-activated Form of an application.
getActiveFrame This method returns the currently-activated frame of an application.
getEventHandler This method returns a specific handler from a certain event of an application by specifying the index number of the handler.
insertEventHandler This method is used to add a handler to a certain event of an application at a specified position.
isExistVariable This method returns the existence of a certain variable defined in the AppVariables field of an application.
lookup This method searches an entire application for the object delivered as an argument and returns the object.
removeEventHandler This method is used to remove a handler in a certain event of an application.
removeEventHandlerLookup This method is used to find a handler in a certain event of an application and remove it.
removeTray This method is used to delete a Tray object registered in the trays property of an application.
removeVariable This method is used to remove a variable from the AppVariables field of an application.
setEventHandler This method is used to change the first handler in a certain event of an application.
setEventHandlerLookup This method finds a function and uses it to replace the first handler in a certain event of an application.
setVariable This method is used to add a variable to the AppVariables field of an application. This method is also used to set the value of a variable defined in the AppVariables field of an application.
trace This method is used to leave a trace log.
transaction This is a method of calling a service to update the data in the Dataset defined in the application area and executing the callback function when the transaction is completed.
Event
Name Description
onafteruserconfirm This event occurs after a user confirmation dialog is closed.
onbeforeexit This event occurs before an application is closed.
onbeforeuserconfirm This event occurs before a user confirmation dialog is displayed.
onduplicateexecution This event is triggered by the duplicate execution of an application that does not permit duplicate execution.
onerror This event occurs when an error occurs in a transaction conducted in an Application.
onexit This event occurs when an application is closed.
onload This event occurs after all the objects and data of an application are loaded.
onloadforms This event occurs after all the forms connected to a ChildFrame are loaded.
onloadingappvariables This event occurs after all the defined objects are created in the AppVariables field of an application.
onnotification This event occurs when a mobile device receives a notification about Nexacro.
