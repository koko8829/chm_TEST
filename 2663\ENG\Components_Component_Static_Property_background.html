Property Type

Description 
The attribute to set the background area of the Static.
Syntax 
Static.background[= strbackground]
Setting Syntax 
strBackground ::= [<bg-image> | <linear-gradient>] [<background-origin> [<background-clip>]] [<background-color>]

<bg-image> ::= 'none' | <imageurl> [<repeat-style>] [<position>]
   <imageurl> :
 
* XCSS
background : #ffffaa;
background : url('./images/smiley.gif') no-repeat center center #ffffaa;
background : url('./images/smiley.gif') no-repeat center center /auto #ffffaa;
background : url('./images/smiley.gif') no-repeat center center /15px 15%
<bg-image> 
This is a background image set with the format "'none' | <imageurl> [<repeat-style>] [<position>]" 

An image will be displayed upon the color specified by the <background-color> option.

If you specify this value as "none", no image will be displayed.
The <imageurl> option is used to specify an image.
The <repeat-style> option is used to specify the direction of displaying an image repeatedly.
The <position> option is used to specify the position of displaying an image.

You cannot set gradients and a background image simultaneously.
<imageurl> 
This value sets a background image with the format "'URL(' <Theme-Image> | <Web-Image> ')'".

It sets an image from the Web or a theme file with the format "URL ( [path of an image] )".
<Theme-Image> 
You can specify an image declared in a theme file with the format "theme://images\image name".

The relevant image must be defined in the theme file.
<Web-Image> 
You can specify an image located in the Web with the format "http://path/image name" or "https://path/file name".

You must have a Web-enabled environment.
<repeat-style> 
This is the direction of displaying an image repeatedly.

If you specify "repeat-x," an image will be repeated in the horizontal direction.
If you specify "repeat-y," an image will be repeated in the vertical direction.
If you specify "repeat," an image will be repeated in both the horizontal direction and the vertical direction.
If you specify "no-repeat," an image will not be repeated.

If you specify no value, the "repeat" option will apply.
<horizontal-position> 
This is the horizontal position of displaying an image.

If you specify "left," the left side of an image will be aligned to the left side of a Static.
If you specify "center," the center of an image will be aligned to the center of a Static.
If you specify "right," the right side of an image will be aligned to the right side of a Static.
If you specify a percentage as a value of <pos-percentage>, the image will be positioned as the value expressed in ratio.


If you do not specify <horizontal-position> while specifying <vertical-position>, the value of <horizontal-position> will be set to "center" automatically.
If you do not specify both <horizontal-position> and <vertical-position>, the value of <horizontal-position> will be set to "left" automatically.
<vertical-position> 
This is the vertical position of displaying an image.

If you specify "top," the top of an image will be aligned to the top of a Static.
If you specify "center," the center of an image will be aligned to the center of a Static.
If you specify "bottom," the bottom of an image will be aligned to the bottom of a Static.
If you specify a percentage as a value of <pos-percentage>, the image will be positioned as the value expressed in ratio.

If you do not specify <vertical-position> while specifying <horizontal-position>, the value of <vertical-position> will be set to "center" automatically.
If you do not specify both <horizontal-position> and <vertical-position>, the value of <vertical-position> will be set to "top" automatically.
<pos-percentage> 
This value indicates the horizontal/vertical position of displaying an image, expressed in ratio.

If you specify 0% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "left".
If you specify 50% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "center".
If you specify 100% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "right".
If you specify 0% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "top".
If you specify 50% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "center".
If you specify 100% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "bottom".

If you specify a number smaller than 0 or bigger than 100, the position will be a virtual point located outside of the Static.

If you specify this value, you cannot omit the unit "%".
<background-size> 
Set the size of the image to be displayed in the background.
- auto: Keeps the original image size.
- cover: Set to the largest so that there is no empty space in the background, as long as the image proportions are preserved.
  Longer widths or heights may be cut off when displayed.
- contain: Set to the largest as long as the image is not cut off and the proportions are preserved.
- <length> | <percentage>: Set the image's width and height, or the ratio to component. 
  If only the width value is set, the height is automatically processed according to the ratio of the original image.
  The height value cannot be set alone. 
  Negative values cannot be set.
<linear-gradient> 
The gradient to be displayed in the background is set in the form of "'linear-gradient(' [<angle> , ] <color-stop> ')'".

If the <background-color> value is set, the gradient is displayed over the background color.

The gradient direction and color can be set in the form of "linear-gradient ( [direction, color value, color value, ...] )".

The gradient and the background image cannot be set at the same time.
<angle> 
Set the display direction of the gradient.

When “to left" is set, the gradient is displayed from the right to the left.
When "to right" is set, the gradient is displayed from the left to the right.
When “to top" is set, the gradient is displayed from the bottom to the top.
When “to bottom" is set, the gradient is displayed from the top to the bottom.
When “to left top" is set, the gradient is displayed from the bottom right to the top left.
When “to left bottom" is set, the gradient is displayed from the top right to the bottom left.
When "to right top" is set, the gradient is displayed from the bottom left to the top right.
When "to right bottom" is set, the gradient is displayed from the top left to the bottom right.

- It is applied as "to bottom" if a value is not set.
<color-stop> 
The color of the start, end, and midpoint of the gradient is set in the form of <color-stop> ::= <startcolor-stop> [<listcolor-stop>] <endcolor-stop>.

The location of the start and end point is determined according to the direction set at the <angle> value.
For example, if the <angle> value is "to left," the right position of the Static becomes the start point and the left position becomes the end point.
              If the <angle> value is "to right", the left position of the Static becomes the start point and the right position becomes the end point.

One or more midpoint colors of the gradient can be set in the <listcolor-stop>, and it is displayed evenly if the <percentage> value is omitted.
<startcolor-stop> 
The color of the start point of the gradient is set in the form of "<color> [<percentage>'%']".

The location of the start point is determined according to the direction set in the <angle> value.
When the <percentage> value of the start point is set as “0” or higher, the same color is displayed from the “start point to set value”
<endcolor-stop> 
The color of the end point of the gradient is set in the form of "<color> [<percentage>'%']".

The location of the end point is determined according to the direction set in the <angle> value.
When the <percentage> value of the end point is set as “100” or lower, the same color is displayed from the “set value to end point”.
<listcolor-stop> 
The color of the midpoint of the gradient is set in the form of "<color> [<percentage>'%"] [,<listcolor-stop>]*".

Multiple colors can be set for the midpoint to be applied in between the start and end points by separating it with “,”.
The <percentage> value of the color declared later should be greater than the <percentage> value of the color declared earlier.
<color> 
The gradation color is set by the color name or color code.
<percentage> 
The location where the color to be applied is set in the <color> by the ratio value.

The start point is determined according to the <angle> value applied as “0%”.
The end point is determined according to the <angle> value applied as “100%”.

When a value smaller than 0 or greater than 100 is set, it is applied to a virtual location outside the Static's area.
For example, when set as “150%”, the <color> value is applied to the position at a position deviating from the end point by half the distance from the start point to the end point

If the <percentage> value is omitted, the <color> value is applied evenly.
The "%" unit cannot be omitted when setting the value.
<background-origin> 
Set the reference area to be processed as the background.

The area including the Border is processed as the background when setting the “border-box”.

The inner area of the Border is processed as the background when setting the "padding-box".
When the "padding-box" is set, the start and end points of the gradient and the repetitive reference position of the background image are processed inside the Border.

It is applied as the "padding-box" if the value is not set.
<background-clip> 
Set the area where the background will be actually displayed.

The background is displayed in the area including the Border when setting the “border-box”.
The background is displayed in the inner area of the Border when setting the "padding-box".

If the value is not set, it is applied as the same value set in the <background-origin>.
If the <background-origin> value is also not set, it is applied as the "border-box".
<background-color> 
The color to be displayed in the background is set by the color name or color code.

If the <bg-image> or <linear-gradient> value is set, the background color is displayed below the background image or the gradient.
If the image applied to <bg-image> has a part applied as transparent or if the image is smaller than the Static area, the background color is displayed in that area.

The gradient and the background image cannot be set at the same time.
<NamedColor> 
Set a color name defined in advance such as “red” or “blue” as strings.
<NumericColor> 
This is the color code for the color to be displayed.

The color code can be set with a 6-digit hexadecimal value starting with the "#" symbol. (ex-"#FF00FF")
The color code can be set using the “rgb()” method that takes red, green, and blue values as factors. (ex-"rgb(255,0,255)")
The color code can be set using the “rgba()” method that takes red, green, blue, and alpha values as factors. (ex-"rgba(255,0,255,0.3)")
The color code can be set using the"hsl()" method that takes hue, saturation, and lightness values as factors. (ex-"hsl(120,50%,50%)")
The color code can be set using the"hsal()" method that takes hue, saturation, lightness, and alpha values as factors. (ex-"hsla(120,50%,50%,0.8)")
Remark 
- It is set as undefined and operated as "transparent" if the background attribute value is not set.

- The user prefix defined in the Services of the TypeDefinition area cannot be used as the image path in the theme or XCSS.
   The user prefix can be used when set as the attribute value in the script or the Nexacro Studio.

- The criterion area and the display area of the background is set as <background-origin> and <background-clip>, respectively.

- The <position> value should be set as "center" or "center center" to locate the background image to the center.

- The image can be displayed on the background color if both the background image and background color are set.
   The gradation is displayed on the background color if both the gradation and background color are set.

- The gradation and the background image cannot be set at the same time.
   A malfunction may occur when setting both values at the same time.
