Description 
This property is used to define the format of an input.
Syntax 
MaskEdit.format [=strFormat]
Setting Syntax 
strFormat ::= <strSNummask> | <strStringmask>

<strSNummask> ::= [<strSignmask>] <strNumbermask>
 
this.MaskEdit00.format = "@@@@";
this.MaskEdit00.format = "###,000.00##";
this.MaskEdit00.format = "999,000.0099";
<strSignmask> 
It specifies a symbol mask to add to the beginning of a number mask. This option is applicable when the type property is set to "number".

1. Symbol mask
  "-" : allows only a minus sign for a negative number
  "+" : allows only a plus sign for a positive number
  "!" : not allows any sign
<strNumbermask> 
This option is applicable when the type property is set to "number".

If you specify a number mask, the number of digits in the fractional part will be restricted, unlike the integer part.
If you do not specify a number mask, there will be no restriction on the number of digits for both the integer part and the fractional part. 
If you specify characters not defined as a mask, those will be ignored.

1. Masks for number inputs
  "#": You can enter a digit (0 to 9).
        If this mask is placed between a "0" mask and a decimal separator, it will function same as a "0" mask.
        It will enter a blank if there is no value or the value is 0.
  "9": You can enter a digit (0 to 9). 
        If this mask is placed between a "0" mask and a decimal separator, it will function same as a "0" mask.
        If there is no value, it will enter a blank. If the value is 0, it will display 0.
  "0": You can enter a digit (0 to 9).
        The relevant digit will show 0 if there is no input to the digit.
        It will show 0 if there is no value or the value is 0.
        The relevant MaskEdit will not display a number when it is being edited.
  ".": Decimal separator
        If there are more than one decimal separator mask, the left end one will apply while the rest is ignored.
        If there is only a decimal separator mask, the relevant cell will function as if no mask is set.
       A decimal separator can adopt a character other than a dot (.) according to the system locale.
  ",": Comma
        A comma mask applies only on the screen, not to the value property.
        If you set a comma mask, thousands separators will be employed to the integer part in a way that matches the system locale, no matter where you put the mask.
        If the system locale is not set, the delimiter commonly occurs every three digits in the integer part.
         A thousands separator can adopt a character other than a comma (,) according to the system locale.

ex) The following examples show the results of applying each mask when the value is 1234.567.
 a. "09999.99"          // The result is "01234.56".
 b. "#9900.0090"      // The result is "1234.5670".
 c. "9,999.##"         // The result is "1,234.56".

ex) The following examples show the results of applying each mask when the value is 0.
 a. "##.#"          // The result is "".
 b. "99.9"          // The result is "0".
 c. "00.0"          // The result is "00.0".

ex) The following examples show the results of applying each mask when the value is 0.1.
 ㄱ. "#.#"           // The result is ".1".
 ㄴ. "9.9"           // The result is ".1".
 ㄷ. "0.0"           // The result is "0.1".
<strStringmask> 
This option is applicable when the type property is set to "string".

1. Masks for string inputs
  "@": all Ascii characters 
          Input of Korean and other non-English characters is impossible.
  "#": decimal numbers (0-9)
  "*": lower case alphabet letters (a-z) and upper case alphabet letters (A-Z)
  "9": lower case alphabet letters (a-z), upper case alphabet letters (A-Z) and decimal numbers (0-9)
  "A": upper case alphabet letters (A-Z)
  "a": lower case alphabet letters (a-z)
  "Z": upper case alphabet letters (A-Z) and decimal numbers (0-9)
  "z": lower case alphabet letters (a-z) and decimal numbers (0-9)

2. Use of single quotations 
A single quotation(') will not be display while the text between single quotations will be displayed without being treated as a mask.
  ex) "@'A'@"is displayed as "_A_" on the screen.

3. Use of \
The symbol \ displays following characters as they are. It is used to display a single quotation or symbol \ itself.

The use of a symbol \ varies depending on where it is used: a dataset column, the Form design area and the script area.
If you use it in a dataset column or the Form Design area, \ will be automatically converted into \\.
By contrast, such automatic conversion will not be conducted in the scrip area. (However, it will be conducted when the component refers to the mask property.) 
  ex) If you specify "@\\@" in the Form Design area, it will be converted into "@\\\\@" and displayed as "_\_" on the screen.
       If you specify "@\'@" in the Form Design area, it will be converted into "@\\'@" and displayed as "_'_" on the screen.
       If you write the code format = "@\\\\@" in the script, it will be displayed as "_\_" on the screen.
       If you write the code format = "@\\'@" in the script, it will be displayed as "_'_" on the screen.

4. PassWord
If you surround mask characters with curly brackets ({}), the characters will be replaced with asterisks (*) when they are displayed.
This option is used to treat input characters as a password.
  ex) If the mask property is set to "@@@{999}" and the value is 123456, the displayed result is "123***".

5. Excluding mask characters
Characters specified in the mask property can be overwritten when a value contains characters that do not follow the defined masking pattern. Such an instance will occur if the below two conditions are met.

  A. The value property is the same as the mask property in length.
  B. Non-mask characters are placed in the same positions both in the mask property and in the value property.

When the above two conditions are satisfied, the cell will ignore input characters that do not follow the defined masking pattern.

ex ) A mask is defined as "@@@@-@@-@@".
 a. If the value is 20060607, the result will be displayed as 2006-06-07.
 b. If the value is 200606, the result will be displayed as 2006-06-__.
 c. If the value is 2006-06, the result will be displayed as 2006--0-6_.
 d. If the value is 2006-06-07, the result will be displayed the same as 2006-06-07.
 e. If the value is 2006-06-07 13, the result will be displayed as 2006--0-60.

As above, only in the "d" case, which satisfies A and B condition, the input characters overwrite the character defined in the mask property.
Remark 
- When the type property is set to "number", you can enter both a minus sign and a plus sign if you omit the value of the  option.

- When the type property is set to "number", there will be no restriction on the number of digits for both the integer part and the fractional part if you omit the value of the <strNumbermask> option.

- When the type property is set to "string", an input cannot be made if you omit the value of the <strNumbermask> option.


◆ Restrictions related to mobile devices

- While a keyboard used under Microsoft Windows cannot distinguish the won sign (\) and the backslash (＼), a mobile keypad can do.
  If you want to show "\" through the mask property, therefore, the correct syntax will be "＼\".

- Depending on the value of the type property, the keypad displayed during text editing is different.
   If the value of the type property is "number", the keypad of type number
   If the type property value is "string", the keypad of the corresponding type according to the mask string set in the format property value.

◆ Restrictions in the Android NRE

- Some virtual keyboards automatically convert input in Korean to English characters.
   In this case, the provided input is treated as English language.
