trackPopup
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method is used to open a PopupMenu in a certain position.
Syntax 
PopupMenu.trackPopup( nXpos, nYpos [,strAlign [,bCapture]] )

strAlign ::= <hAlign> ' ' <vAlign>
<hAlign> ::= 'left' | 'center' | 'right'
<vAlign> ::= 'top' | 'middle' | 'bottom'
Parameters
Parameters Type Description
nXpos Number 
This is a number representing the x-coordinate of the position where the PopupMenu is opened. You should calculate this value based on the top left side of the MainFrame.
nYpos Number 
This is a number representing the y-coordinate of the position where the PopupMenu is opened. You should calculate this value based on the top left side of the MainFrame.
<hAlign> String 
This parameter determines the horizontal alignment of the PopupMenu based on the value of the nXpos parameter.

If you specify "left", the right side of the PopupMenu will be aligned with the position defined by nXpos.
If you specify "center", the center of the PopupMenu will be aligned with the position defined by nXpos.
If you specify "right", the left side of the PopupMenu will be aligned with the position defined by nXpos.

The default value is "right".
<vAlign> String 
This parameter determines the vertical alignment of the PopupMenu based on the value of the nYpos parameter.

If you specify "top", the bottom of the PopupMenu will be aligned with the position defined by nYpos.
If you specify "middle", the middle of the PopupMenu will be aligned with the position defined by nYpos.
If you specify "bottom", the top of the PopupMenu will be aligned with the position defined by nYpos.

The default value is "bottom".
bCapture Boolean 
This parameter determines whether to offer the Capture feature.

For "true", the PopupMenu will appear with the Capture feature.
For "false", the PopupMenu will appear without the Capture feature.

The default value is "true".
Sample Call:
this.PopupMenu00.trackPopup( 10, 10 );
this.PopupMenu00.trackPopup( 10, 10, "right bottom" );
this.PopupMenu00.trackPopup( 10, 10, "right bottom", true );
Return
None
Remark 
- You should combine the values of the <hAlign> and <vAlign> parameters with a whitespace between them to compose the value of the strAlign parameter.

- If this method is executed, the visible property of the PopupMenu will be set to "true" automatically.

- If a PopupMenu does not have the Capture feature, focus will move to another component immediately when you select the component while the PopupMenu will disappear.
 If a PopupMenu has the Capture feature, focus will not move to another component immediately when you select the component. The first selection will make the PopupMenu disappear and then the second selection will move the focus.

- If a title bar is displayed in the Nexacro Runtime Environment, the inner Form will be moved downward as much as the height of the title bar.
 Accordingly, the value of the nYpos parameter can indicate different positions in the Web Runtime Environment and in the Nexacro Runtime Environment.
