Description 
The Radio is a type of button, which is a component that allows you to select one from many round-shaped options. 

If one item is selected by mouse or keyboard, it outputs the selected item only as selected, and the rest of the items as unselected.
The Radio can display the text related to the selected item on the left or right side.
Remark 
- The selected item can be edited manually with a script.
  The shape of the text displayed on the screen can be edited by the font attribute.

- The Radio component can be automatically composed through the InnerDataset or composed using the Contents.
  When using the InnerDataset, the screen composition of the Radio is operated automatically according to the attribute value of the Radio.
  When composed of the Contents, the user must arrange the radio manually.

- The code value and data value of the Radio can be bound with a specific Column value of the DataSet by binding the DataSet.
  The assigned column value of the current row of the DataSet becomes the selected item of the Radio,
  and the Column value of the Dataset is immediately edited if the user selects one item by mouse or keyboard.
Structure
Contents Sizing 
The below table describes how to size a Radio.  - The below descriptions are true when the fittocontents property of the Radio is set to "none".
 
Radio Component
Width 
The value of the width property of the Radio is applied.
Height 
The value of the height property of the Radio is applied. 
radioitem Control
Left 
The left property of a radioitem control is calculated based on the position of the radioitem control, which is determined based on the rowcount,columncount and direction properties. If the rowcount and columncount properties are set at -1, the left property of one radioitem control will be calculated by factoring in the widths of other radioitem controls.
Top 
The top property of a radioitem control is calculated based on the position of the radioitem control, which is determined based on the rowcount,columncount and direction properties.
Width 
The width of a radioitem control is calculated by dividing the width of the Radio by the number of columns, which is determined based on the rowcount, columncount and direction properties. If the rowcount and columncount properties are set at -1, the width of a radioitem control will be the icon width plus the text width. The internal contents will be hidden from sight if the width of a radioitem control is smaller than the internal contents.
Height 
The height of a radioitem control is calculated by dividing the height of the Radio by the number of rows, which is determined based on the rowcount, columncount and direction properties. The internal contents will be hidden from sight if the height of a radioitem control is smaller than the internal contents.
 
icon
Left 
The value of the left property is calculated by factoring in the -nexa-icon-position,-nexa-text-align and -nexa-vertical-align properties.
Top 
The value of the top property is calculated by factoring in the -nexa-icon-position,-nexa-text-align and -nexa-vertical-align properties.
Width 
The width of the icon image is applied.
Height 
The height of the icon image is applied.
 
text
Left 
The value of the left property is calculated by factoring in the -nexa-icon-position,-nexa-text-align and -nexa-vertical-align properties.
Top 
The value of the top property is calculated by factoring in the -nexa-icon-position,-nexa-text-align and -nexa-vertical-align properties.
Width 
The width is calculated automatically according to the text length.
Height 
The height is calculated automatically according to the text size.

Width 
The width of the Radio is calculated by adding widths of all radioitem controls of each column.
 
radioitem Control
Left 
The left property of a radioitem control is calculated by factoring in the following determinants: number of radioitem controls; widths of radioitem controls; and rowcount, columncount and direction properties.

The top property of a radioitem control is calculated by factoring in the following determinants: number of radioitem controls; heights of radioitem controls; and rowcount, columncount and direction properties.
Width 
The width of the radioitem control is calculated by adding the widths of the icon and text. If the relevant column contains another radioitem control that is wider than the current control, the bigger width will be applied.

Basic Key Action
Action Key Description
Move focus Tab 
The focus moves to the next component according to the tab movement order.
Select item while moving between items Left/Right arrow key 
Move to the item to select in order.
The item is changed to the selected state and the changed value is reflected in the value property.
Accessibility Key Action 
This is a key action to be applied when the accessibility function is activated.
Action Key Description
Move focus Tab 
The focus moves to the next component according to the tab movement order.
Move selection box Up/Down arrow key 
A selection box will move between items. An item that receives a selection box will not be selected automatically unless you press the space bar. If you press the up or down arrow key in the first or last item respectively, a selection box will move to the previous or next component according to the tab order respectively.
Select item while moving between items Left/Right arrow key 
Focus will move between items. An item will be selected when it receives focus.
Determine item Space 
Pressing the space bar will determine a selected item and apply it to the value property.
Constructor 
Radio.init( strName, vLeft, vTop , vWidth, vHeight [, vRight, vBottom, [vMinWidth, [vMaxWidth, [vMinHeight, [vMaxHeight]]]]] )
Parameters Type Description
strName String 
It sets the ID of a Radio in the form of a string.
vLeft String, Number 
It specifies the value of the left property of a Radio in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the right property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vTop String, Number 
It specifies the value of the top property of a Radio in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the bottom property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vWidth String, Number 
It specifies the value of the width property of a Radio in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vHeight String, Number 
It specifies the value of the height property of a Radio in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vRight String, Number 
It specifies the value of the right property of a Radio in pixel or percent (%).

This parameter will be ignored if both the vLeft and vWidth parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the left property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vBottom String, Number 
It specifies the value of the bottom property of a Radio in pixel or percent (%).

This parameter will be ignored if both the vTop and vHeight parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the top property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vMinWidth String, Number 
It specifies a Radio's minimum width that can be displayed in the Form.
vMaxWidth String, Number 
It specifies a Radio's maximum width that can be displayed in the Form.

If this value is smaller than vMinWidth, it will be replaced by the value of vMinWidth.
vMinHeight String, Number 
It specifies a Radio's minimum height that can be displayed in the Form.
vMaxHeight String, Number 
It specifies a Radio's maximum height that can be displayed in the Form.

If this value is smaller than vMinHeight, it will be replaced by the value of vMinHeight.

var objComp = new Radio();

objComp.init( "Radio00", 30, 120, 196, 46 );
objComp.init( "Radio00", 30, 120, 196, 46, null, null );
objComp.init( "Radio00", null, null, 300, "400px", "80%", 300 );
objComp.init( "Radio00", 0, 0, 200, 100, null, null, 300, 500, 200, 500 );
objComp.init( "Radio00", "Radio22:10", 300, null, null, "Radio33:10", "20%", 300, 500, 200, 500 );
 
// Create Object  
var objRadio = new Radio();  
objRadio.init("Radio", 30, 120, 196, 46, null, null);
objRadio.init("Radio", null, null, 300, "400px", "80%", 300);
or
var objRadio = new Radio("Radio", 30, 120, 196, 46, null, null);
var objRadio = new Radio("Radio", null, null, 300, "400px", "80%", 300);
 
// Add Object to Parent Form  
this.addChild("Radio", objRadio); 
 
// Insert Object to Parent Form  
this.insertChild(1, "Radio", objRadio); 
 
// Remove Object form Parent Form  
this.removeChild("Radio"); 
 
// Show Object  
objRadio.show(); 
 
// Destroy Object  
objRadio.destroy(); 
objRadio = null;
