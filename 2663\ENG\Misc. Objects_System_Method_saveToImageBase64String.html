saveToImageBase64String
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
The method to return the image of the selected component as an encoded string with Base 64.
Syntax 
system.saveToImageBase64String( objComponent [, bWholeForm [,strImageType [,strOptions]]] )
Parameters
Parameters Type Description
objComponent Object 
This is an object representing a component whose image will be printed.
bWholeForm Boolean 
This parameter is applicable only when the component is a Form.

If you specify "true", the entire Form including the area out of sight will be printed as an image.
If you specify "false", only the area of the Form within sight of the current screen will be printed as an image.

The default value of this parameter is "false".
strImageType String 
Set the image file format for output.
Set the default value as "JPG".

"BMP", "BMP_MONO", "JPG", "TIF", "TIF_24", "TIF_MONO", "GIF", "PNG" formats can be set.
strOptions String 
Set the image compression rate, image compression method, and DPI resolution of the image file to be output.

The compression rate is supported only when the image format is “JPG,” and it is set in the “Compress:compression ratio” format.
The compression ratio is set as an integer value in the range of "0~100".
When the image format is “JPG” and the value is not set, the value is applied as 50.

The compression method is supported only if the image format is “TIF” and “Compress:compression type” is the setting format.
Set one of "NONE", "LZW", "CCITT3" or "CCITT4" as the compression type.
When the image format is “TIF” and the value is not set, “LZW” is applied.

The DPI resolution is set to "ResolutionX:Resolution" format for horizontal and "ResolutionY:Resolution" format for vertical.
The resolution is set to an integer value.
Both horizontal and vertical values are applied as 96 if the resolution is not set.

Separate each option with ";” like "Compress:95;ResolutionX:300."
Sample Call:
var strResult;

strResult = system.saveToImageBase64String( this.Button00 );
strResult = system.saveToImageBase64String( this, true );
strResult = system.saveToImageBase64String( this.Div00.form, true );
strResult = system.saveToImageBase64String( this.Div00.form, true, "TIF_MONO","Compress:CCITT3;ResolutionX:300;ResolutionY:300" );
Return
Type Description
String 
- It returns the image of the component by the string encoded as Base64.
  An empty string is returned when failing to encode.
Remark 
- The image format should be “PNG” to keep the alpha channel value to save the translucent value.
   However, some deformation may occur on the image if the background color is not “black” due to the characteristics when processing the alpha channel value of the “PNG” file.

- The overall characteristic by image format is as follows although there are differences according to the feature of the component.
   Capacity: BMP > TIF > PNG > JPG(100) > GIF > JPG(0)
   Quality: (BMP = PNG) > TIF > JPG(100) > GIF > JPG(0)

- The ColorDepth is saved as 1bit monochrome unconditionally if the image compression method is “"CCITT3" or "CCITT4”.

- The "BMP_MONO" and "TIF_MONO" image formats are saved as 1bit black and white monochrome images.

- The "TIF_24" image format is saved as a 24bit RGB image.
