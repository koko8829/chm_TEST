Description 
The Calendar is a component to receive date and time input.
Remark 
- It provides functions related to input more than the screen output of date and time.

- The form of the input area and pop-up calendar, the monthly calendar form, and the form of the input area and spin button can be selected by using the type attribute value.

- It has a built-in function to examine the consistency of the input date.

- You can give an effect to a specific date using the innerdataset attribute.

- You can set the format of the date to be input using the editformat attribute value.
   - You can set the format of the date to be displayed using the dateformat attribute value.
   You can set a date in a different form by country when the attributes of the editformat and dateformat are set as "LONGDATE" and "SHORTDATE".

- You can input conveniently using the spin button provided for the screen or the up/down key of the keyboard.
   The date value changes consecutively if the LButtonDown state is kept on the spin button displayed on the Calendar and the pop-up calendar.

- The date on the pop-up calendar can be selected by moving down&up with the left mouse button.

- The pop-up calendar may open and then close immediately if the dropbutton and the pop-up calendar are overlapped.
   It’s a problem when the click position is overlapped, but not a bug, so you need to change the setting value so that the dropbutton and the pop-up calendar are not overlapped.

- The time should be input manually from the edit area if the mask characters related to time are set in the editformat attribute value.
   The time must be entered in 24-hour format when input, and 12-hour format can be used for the display.

- If an invalid value is entered while entering the date value to the calendaredit area manually, the consistency should be examined according to the following rules and the calendar value should be reflected.

(1) When the existing value is a leap day
If the existing value is a leap day and you enter a year value other than a leap year, the date value is reflected as February 28th.
For example, if the existing value is "2020-02-29" and you enter "2025" as the year value, the date value will be reflected as "2025-02-28".

(2) Handling invalid numbers when entering unit-digits
When entering unit-digits in the month or day edit area and an invalid number is entered, the tens-digit is treated as "0" and only the unit-digit is retained.
This is the case if you enter a number between 13 and 19 in the month edit area or a number greater than the maxday in the day edit area.
For example, if you enter "13" in the month edit area, the tens-digit is treated as "0" and the date value is reflected as "03", and the caret moves ahead of the tens-digit in the day edit area.
If you enter "32" in the day edit area, the tens-digit is treated as "0" and the date value is reflected as "02" (in the case of February, entering 3 in the tens-digit is treated as an invalid number.), and the caret moves after the unit-digit in the day edit area.

(3) Handling invalid numbers when entering tens-digits
When entering tens-digits in the month or day edit area and an invalid number is entered, the tens-digit is treated as "0" and only reflected on the unit-digit.
For example, if the existing value is "2020-11-11" and you enter "2" in the tens-digit of the month edit area, the date value will be reflected as "2020-02-11"., and the caret moves ahead of the tens-digit in the day edit area.
If you enter "4" in the tens-digit in the day edit area, the date value will be reflected as "2020-11-04", and the caret moves after the unit-digit in the day edit area.

◆ Mobile restrictions

- In the mobile environment, the date does not change consecutively even if you keep touching the spin button due to a functional problem.

Basic Key Action
Action Key Description
Move focus Tab 
The focus moves to the next component according to the tab movement order.
Open DatePicker Alt + Down arrow key or Option + Down arrow key 
The pop-up calendar (DatePicker) is displayed when the type attribute value is “normal”.
Close DatePicker ESC 
When the type property value is "normal", it closes the open pop-up calendar (DatePicker).
Change date Up/Down arrow key 
With the type property set to "spin", pressing up/down arrow key will change a year, month or day value depending on the position where a caret is placed. 

If the year, month or day value reaches the largest/smallest value, pressing another up/down arrow key will change the value to the one on the other end (largest to smallest or vice versa).
Determine date Enter 
With the type property set to "normal" or "monthonly", pressing the enter key will apply the date selected from the pop-up calendar (date picker) to the value property.
Move focus between dates in DatePicker Arrow keys of four directions 
Focus will move between dates in the date picker. 
You must press the enter key to determine a selected date. 

If you press the up/left arrow button in the first date or down/right button in the last date, the month will change.
Move focus between dates in DatePicker Ctrl + left/right arrow key 
The month will change in the date picker.
Move focus between dates in DatePicker Ctrl + up/down arrow key 
The year will change in the date picker.

Constructor 
Calendar.init( strName, vLeft, vTop , vWidth, vHeight [, vRight, vBottom, [vMinWidth, [vMaxWidth, [vMinHeight, [vMaxHeight]]]]] )
Parameters Type Description
strName String 
It sets the ID of a Calendar in the form of a string.
vLeft String, Number 
It specifies the value of the left property of a Calendar in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the right property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vTop String, Number 
It specifies the value of the top property of a Calendar in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the bottom property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vWidth String, Number 
It specifies the value of the width property of a Calendar in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vHeight String, Number 
It specifies the value of the height property of a Calendar in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vRight String, Number 
It specifies the value of the right property of a Calendar in pixel or percent (%).

This parameter will be ignored if both the vLeft and vWidth parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the left property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vBottom String, Number 
It specifies the value of the bottom property of a Calendar in pixel or percent (%).

This parameter will be ignored if both the vTop and vHeight parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the top property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vMinWidth String, Number 
It specifies a Calendar's minimum width that can be displayed in the Form.
vMaxWidth String, Number 
It specifies a Calendar's maximum width that can be displayed in the Form.

If this value is smaller than vMinWidth, it will be replaced by the value of vMinWidth.
vMinHeight String, Number 
It specifies a Calendar's minimum height that can be displayed in the Form.
vMaxHeight String, Number 
It specifies a Calendar's maximum height that can be displayed in the Form.

If this value is smaller than vMinHeight, it will be replaced by the value of vMinHeight.
