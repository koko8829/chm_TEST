Property Type

Description 
This property is used to color the inside area of a GraphicsPath.
Syntax 
GraphicsStock.fillstyle[= strFillStyle]
Setting Syntax 
strFillStyle ::= <color> | <linear-gradient>

<color> ::= <NamedColor> | <NumericColor>

<linear-gradient> ::= 'linear-gradient(' [<angle> , ] <color-stop> ')'
<angle> ::= 'to left' | 'to right' | 'to top' | 'to bottom' | 'to left top' | 'to left bottom' | 'to right top' | 'to right bottom'
<color-stop> ::= <startcolor-stop> [',' <listcolor-stop>] ',' <endcolor-stop>
<startcolor-stop> ::= <color> [<percentage>'%']
<listcolor-stop> ::= <color> [<percentage>'%'] [',' <listcolor-stop>]*
<endcolor-stop> ::= <color> [<percentage>'%']
 
var objGObject = new nexacro.GraphicsPath();

objGObject.fillstyle = "red";
objGObject.fillstyle = "rgb(255,255,0)";
objGObject.fillstyle = "#ffffaa";
objGObject.fillstyle = "linear-gradient(red, yellow)";
objGObject.fillstyle = "linear-gradient(to right, red, yellow)";
objGObject.fillstyle = "linear-gradient(to right, red 50%, yellow)";
objGObject.fillstyle = "linear-gradient(red, yellow 25%, green 50%, blue 75%, indigo)";
objGObject.fillstyle = "linear-gradient(to right, red, orange, yellow, green, blue, indigo, violet)";
<color> 
This is a color that fills the inside of the GraphicsPath. You can specify the value with a color name or color code.
<NamedColor> 
This is a predefined color name in the form of a string like "red" and "blue".
<NumericColor> 
This is a hexadecimal value that begins with "#" and comprises the following 6-digit value (eg. #FF00FF). This hexadecimal value indicates the color inside of the GraphicsPath.

You can specify a color with the rgb() method whose arguments are r, g and b values. (eg. rgb(255,0,255))
<linear-gradient> 
This value is used to create gradients inside the GraphicsPath.

You can specify the direction and colors of gradients with the format "linear-gradient ( [direction, color1, color2, ...] )".
You can use more than one colors through the <listcolor-stop> value.
<angle> 
This value indicates the direction of gradients.

If you specify "to left," gradients will run from right to left.
If you specify "to right," gradients will run from left to right.
If you specify "to top," gradients will run from bottom to top.
If you specify "to bottom," gradients will run from top to bottom.
If you specify "to left top," gradients will run from bottom right to top left.
If you specify "to left bottom," gradients will run from top right to bottom left.
If you specify "to right top," gradients will run from bottom left to top right.
If you specify "to right bottom," gradients will run from top left to bottom right.

If you specify no value, the "to bottom" option will apply.
<color-stop> 
This value defines colors for the beginning, middle and ending points of the gradients.

The beginning and ending points are determined according to the <angle> value.
You can derive the beginning and ending points from the value of <angle>, which indicates the direction of the gradients.

Assuming that <angle> is specified as "to left", for example, the right and left sides of a GraphicsPath will be beginning and ending points respectively.
Assuming that <angle> is specified as "to right", for example, the left and right sides of a GraphicsPath will be beginning and ending points respectively.

You can specify more than one color for the middle points of the gradients. The specified colors will be equally distributed if you omit the value of <percentage>.
<startcolor-stop> 
This is the color of the beginning point of the gradients.

A beginning point will be determined according to the value of <angle>.
If you specify the <percentage> of the beginning point as 0 or larger, a same color will be displayed from the beginning point to the specified point.
<listcolor-stop> 
This value defines the colors of the middle points of the gradients.

You can specify multiple colors to apply between the beginning and ending points by dividing them with commas.
<endcolor-stop> 
This is the color of the ending point of the gradients.

An ending point will be determined according to the value of <angle>.
If you specify the <percentage> of the ending point as 100 or smaller, a same color will be displayed from the specified point to the ending point.
<percentage> 
This value defines the positions where the colors specified by <color> will apply. Those positions are specified in percentage.

"0%" indicates the beginning point determined according to the value of <angle>.
"100%" indicates the ending point determined according to the value of <angle>.

This value must range between 0 and 100.
If you omit this value, the value of <color> will apply to a proportionate point.
Remark 
- In the case of a GraphicsPaths and GraphicsPath, this property can be applied only when those paths represent closed forms.
