Description 
The attribute to set the direction in which the items are displayed in Radio.
Syntax 
Radio.direction[= enumDirection]
Setting Syntax 
enumDirection ::= 'horizontal' | 'vertical'
 
this.Radio00.direction = "horizontal";
"horizontal" 
Items will be displayed from the left to the right. If a row is full of items, following items will be laid in the next rows.

The number of items that can be displayed in a row is determined by the columncount property.
"vertical" 
Items will be displayed top to bottom. If a column is full of items, following items will be laid in the next columns.

The number of items that can be displayed in a column is determined by the rowcount property.
Remark 
(1) Actions when setting the default value

- It is applied as “horizontal" if the direction attribute value is not set.
- The radio item is displayed on one row if the attribute value of the columncount, rowcount, and direction is not set (if default value).  

(2) Action according to the direction attribute value when the attribute value of both columncount and rowcount are set

- The rowcount attribute value is ignored if the columncount attribute value is 1 or higher when the direction attribute value is set as "horizontal".
- The columncount attribute value is ignored if the rowcount attribute value is 1 or higher when the direction attribute value is set as "vertical".

(3) Maintaining the current status for wrong actions

The 3 actions below are wrong, but it is not to be modified but to keep the current status to secure the actions of the code written by the existing user. These actions may change after an update, so keep it in mind when writing a new code.

(3-1) The radio item is displayed on one column if the attribute value of the columncount and rowcount is not set and, and only the direction attribute value is set as “vertical”. It is recommended to set the rowcount attribute value as 1 to display the radio item on one column.

(3-2) The radio item is automatically allocated according to the length of the item text when the attribute value of both columncount and rowcount is -1 and the fittocontent attribute value is “none”. At this time, the value set as the direction attribute value is ignored and processed as “horizontal”. It is a wrong action to ignore the direction attribute value, it is maintained as the current status without modification to secure the action of the code written by the existing user.

(3-3) It operates as the following if the columncount attribute value is set as 0 and the rowcount attribute value as -1, or the column count attribute value as -1 and the rowcount attribute value as 0. It is recommended to set the rowcount attribute value as 1 to display the radio item on one column, and to set the columncount attribute value as 1 to display the radio item to display the radio item on one row.
It is recommended to set the columncount attribute value as 0 and the rowcount attribute value as -1 to display the radio item on one column.
- If the columncount attribute value is set as -1, the rowcount attribute value as 0, and the direction attribute value as “vertical”, the radio item is displayed on one column. If only the direction attribution value is set has “horizontal”, the radio item is displayed on one row.
