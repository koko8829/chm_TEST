Description 
This property is used to set a device type, which serves as a base for selecting a screen where an application runs.
Syntax 
Screen.type
Setting Syntax 
type ::= 'desktop' | 'mobile_small' | 'mobile_medium' | 'mobile_large'
"desktop" 
This value can be selected as a current screen when the application is executed in the desktop PC environment.
"mobile_small" 
This value can be selected as a current screen when the application is executed in a mobile device whose display width is 8cm or less in the portrait mode.
"mobile_medium" 
This value can be selected as a current screen when the application is executed in a mobile device whose display width is larger than 8cm and smaller than 13cm or as large as 13cm in the portrait mode.
"mobile_large" 
This value can be selected as a current screen when the application is executed in a mobile device whose display width is larger than 13cm in the portrait mode.
Remark 
- This property can be set only in Nexacro Studio and cannot be modified during execution of the application.

- A screen for running an application is selected from the list through the below process.

  1) Information on the execution environment will be dynamically created, and the information will be used as a base for selecting a screen.
    1-1) If the execution environment is mobile, one of the following values will be selected based on the display's physical width: "mobile_small", "mobile_medium" and "mobile_large".
  2) If the value selected in the Step 1 matches the value specified by the type property, the relevant value will be selected as the screen.
  3) If more than one screens are selected in the Step 2, the value of the specifiedos property will be considered in order to compare itself with the value selected in the Step 1. If there is any matching value, the relevant one will be selected as the screen.
  4) If more than one screens are selected in the Step 3, the value of the specifiedlocale property will be considered in order to compare itself with the value selected in the Step 1. If there is any matching value, the relevant one will be selected as the screen.
  5) If more than one screens are selected in the Step 4, the first one will be selected as the screen.
  
  * The following procedure will be followed if none among the values selected in the Step 1 matches the value specified by the type property. First, Nexacro Platform will attempt to find a screen smaller than the base determined in the Step 1. If there is no smaller one, then Nexacro Platform will attempt to find a bigger screen.
 For example, assume that the base determined in the Step 1 is "mobile_large" and the value of the type property is "desktop". Then, Nexacro Studio will attempt to find "mobile_medium" and "mobile_small" first. If failed, Nexacro Studio will select "desktop" as the screen.

 * This property always applies "mobile_large" to iPad series (including mini1, 2, 3, 4, iPad1, 2, air1, air2, pro) due to the lack of the information on the distinction between devices.

- If the normal process of selecting a screen fails, a default screen will apply.

- In the portrait orientation, "mobile_small", "mobile_medium" and "mobile_large" will be treated as "480x848", "720x1280" and "1080x1920" in display resolution, respectively.
