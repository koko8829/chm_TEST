Description 
This property is used to determine the type of an editable field for a focused Grid cell.
Syntax 
GridCellControl.edittype[= enumEdittype]
Setting Syntax 
enumEdittype ::= 'normal' | 'none' | 'button' | 'checkbox' | 'combo' | 'date' | 'mask' | 'multicombo' | 'radioitem' | 'readonly' | 'text' | 'textarea' | 'tree'
 
this.Grid00.setCellProperty( "body", 0, "edittype", "normal" );
"normal" 
The type of an editable field is determined automatically according to the type of bound data.

The type will be either "text" or "date" depending on the type of a bound Dataset column.
"none" 
The Grid cell will not feature an editable field. 
The cell will remain impossible to edit.
"button'" 
The Grid cell represents a button while bound data is displayed as text in the button. 

If you click or press the space bar on the button, a relevant pseudo-class will be applied.
If the button is displayed with the fillareatype property of the cell applied, the "disabled" pseudo-class will be applied.
"checkbox" 
There is no change in the cell when you click the checkbox. Instead, the bound data will change from 0 to 1 or vice versa.
"combo" 
The cell represents a combo, which presents bound data. 

To display a combo in a cell, you must set the combodataset, combocodecol and combodatacol properties.
The combo displays the values held by the Dataset column specified by combodatacol while the values match each code presented by the Dataset column specified by combodatacol.
When you click the drop-down button on the right side of the cell, the item list will appear.
"date" 
The Grid cell represents a calendar, which presents data according to specified display formats. 

The display formats of days/time are determined by the calendardateformat property of the cell.
"mask" 
The Grid cell represents a MaskEdit. 

You can specify the input format applied to the cell through the maskeditformat property of the Cell.
The cell features a MaskEdit which a value of maskeditformat applies to.

If you want to enter numbers, you must set the maskedittype property to "number".
"multicombo" 
The Cell area is displayed in the MultiCombo type, and the data value of the MultiCombo item that matches the bound data is displayed.

The multicombodataset, multicombocodecol, and multicombodatacol property values must be set to display in the MultiCombo type.
The multicombodatacol value corresponding to the multicombocodecol value matching the bound data is displayed on the screen.
The item list is displayed when you click the pop-up button on the right side of the Cell.
"radioitem" 
The Gird cell area is displayed in the form of a radioitem.

When the cell area is clicked, the selected state is displayed and the bound data is changed to the radioitemcodevalue property value.
"readonly" 
The Grid cell represents a TextArea.

However, you can only read the value of the cell and cannot edit it.
"text" 
The Grid cell represents an Edit.
"textarea" 
The Grid cell represents a TextArea.
"tree" 
This option is applicable only when the displaytype property is set to "tree".
You can collapse/expand a tree node by clicking a tree button displayed in a cell.
Remark 
- The default value of this property is "none".

- If you change the data of a cell, the cancolumnchange event will occur in the bound Dataset, followed by the oncolumnchanged event.
 If the cancolumnchange event returns "false", the oncolumnchanged event will not occur and the pre-change data will be restored.

- The editable field in a cell can be styled with a theme.

- If you want to limit an input to the numeric format when the edittype property is set to "text" or "textarea", you should set the textareainputfilter property.

- If a merged cell contains sub-cells, the edittype properties of each sub-cell will be ignored.
 edittype will be applicable only when a merged cell constitutes a single cell.
