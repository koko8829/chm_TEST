Description 
This event occurs when focus comes into a CheckBox.
Syntax 
onsetfocus(obj:nexacro.CheckBox,e:nexacro.SetFocusEventInfo);
Parameters
Parameters Type Description
obj CheckBox 
Event generated Object.
e SetFocusEventInfo 
Event Object.
Return
None
Remark 
- If focus moves between different applications, a focus-related event will not occur.

- If focus moves to a frame opened by a method like showModal() and open(), a focus-related event will not occur to a CheckBox.
  Moreover, if focus moves to a pop-up opened by the alert() or confirm() method, a focus-related event will not occur to a CheckBox.

- Note that the following situation leads to behavior that changes according to environments: the setFocus() method is called to a CheckBox in the parent window from a modeless window that was opened by the open() method.
 > Nexacro Runtime Environment: 
      the onactivate event will be fired by the activation of the parent Form.
      The focus will move to the parent Form and frames will be arranged.
 > Web Runtime Environment: 
      the parent Form will not be activated due to the restrictions related to web browsers. Therefore, the onactivate event will not occur either.
      While the focus will not move to the parent Form, frames will be arranged nevertheless.
   * What it means by "frames will be arranged": if you move focus to one of the child frames of a FrameSet, the focused frame will top the other frames at the same level.
      Against the backdrop, assume that the focus moves to another frame at the same level. Then, the onsetfocus event will occur in the Form of the newly-focused frame.
      The onsetfocus event will not occur if the focus does not move between frames.
