Description 
This method is used to filter the data loaded onto a Dataset so that only the rows of data that satisfies a given condition is shown.
Syntax 
Dataset.filter([strFilterExpr])
Parameters
Parameters Type Description
strFilterExpr String 
This is a condition with which the data of a Dataset will be filtered.

An existing condition will be revoked if you specify an empty string ("").
If you omit this argument, the value of the filterstr property will be applied.
Sample Call:
this.Dataset00.filter();
this.Dataset00.filter("");
this.Dataset00.filter("dept_cd == 'A1'");
this.Dataset00.filter("amount > 1000 && model_cd = 'A-1'");
this.Dataset00.filter("amount > 1000 || amount < 50 ");
Return
None
Remark 
- When you call this method, the filterstr property of the Dataset will change its value to the value of the strFilterExpr parameter.
 If you do not specify the parameter, the value of the filterstr property will be applied as the argument.

- Calling this method will fire the onrowsetchanged event while the reason property of the relevant EventInfo object will have "REASON_FILTER" as its value.

- After the calling of this method, the rowposition property will have an existing value indicating the row that was selected before the calling of this method. rowposition will have -1 if the property does not have such a value.

- If you call this method from a Dataset whose data is already filtered, this method will filter the entire data of the Dataset again with the condition defined by the strFilterExpr parameter.

- Processing the conditional expression defined by the strFilterExpr parameter must result in either "true" or "false".

- When filtering is conducted, the Grid behaves as if its selecttype property is set to "row". Then, the current row will be selected, firing the onrowsetchanged event.
 At this point, the onrowsetchanged event will occur twice while the reason properties of the relevant EventInfo object will have 40 (stat:row type [select]) and 31 (filter) as values respectively. This behavior--onrowsetchanged occurs twice--is correct operation.
 If the settings of the multi-selection function is automatically changed by the Grid, however, a selection-related event will not occur in the Dataset.
