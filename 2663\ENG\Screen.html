Description 
This is an object that manages the information on screens applicable to an application.
Remark 
- You can set environment-related properties for each screen individually. Examples of such properties are themeid and initvaluefileid.

- Nexacro Platform selects a screen to apply to an application according to the following steps.
  1. Nexacro Platform dynamically creates the information on the environment where an application will run. The information will be used as criteria for choosing a screen.
   1-1. If the environment is a mobile type, the information will represent one of the three modes: "mobile_small", "mobile_medium" and "mobile_large".
  2. The platform will choose a screen if the result of Step1 matches the value of the type property.
  3. If more than one screens are selected as a result of Step 2, the platform will pick up one of them by comparing the result of Step 1 and with the value of the specifiedos property.
  4. If more than one screens are selected as a result of Step 3, the platform will pick up one of them by comparing the result of Step 1 and with the value of the specifiedlocale property.
  5. If more than one screens are selected as a result of Step 4, the platform will pick up the one that is defined first.

 * If no screen is selected in Step2; there is no match when comparing the result of Step 1 and the value of the type property,
   Nexacro Platform will select a screen smaller than the criteria determined by Step1. If even there is no smaller screen, the platform will select a bigger one.
   For example, let us assume a case where the criteria determined by Step1 is "mobile_large" while the value of type property is "desktop" only.
   In this case, Nexacro Platform will choose the "desktop" screen eventually after it searches for the "mobile_medium" and then "mobile_small" screens.

 * This property always applies "mobile_large" to iPad series (including mini1, 2, 3, 4, iPad1, 2, air1, air2, pro) due to the lack of the information on the distinction between devices.

- The zoom feature will be applied automatically according to a selected screen based on the value of the zoomfactortype property.

- In the portrait orientation, the resolutions of the "mobile_small", "mobile_medium", and "mobile_large" screens are 480x768, 640x1024 and 800x1280 respectively.
Property
Name Description
id This property is used to specify the ID of a screen.
specifiedlocale This property is used to specify a locale, which serves as a base for selecting a screen for executing an application.
specifiedos This property is used to specify an OS, which serves as a base for selecting a screen for executing an application.
type This property is used to set a device type, which serves as a base for selecting a screen where an application runs.
userzoom This property is used to determine whether to use the pinch zoom function. This property is applicable when an application whose screen type is "desktop" is loaded onto a mobile web browser.
zoomfactortype This property is used to determine the base for the zoom feature applied when an application is loaded in the mobile environment.
