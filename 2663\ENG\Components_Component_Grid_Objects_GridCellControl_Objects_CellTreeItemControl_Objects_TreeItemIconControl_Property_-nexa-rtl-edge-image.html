Property Type

Description 
This property is used to set an edge image that is displayed inside the border of a TreeItemIconControl. This property is applicable when the rtl property of the TreeItemIconControl is set to "true".
Syntax 
TreeItemIconControl.-nexa-rtl-edge-image
Setting Syntax 
strRtlEgImage ::= "URL(' " <Theme-Image> | <Web-Image> " ')"
 
* XCSS
-nexa-rtl-edge-image : URL("http://localhost/Test.jpg");
-nexa-rtl-edge-image : URL("theme://images/Test.jpg")
strRtlEgImage 
This string indicates an image from the Web or a theme file, specified with the format "URL ('<image location>')".

Among possible values of the -nexa-edge property, only an image will be replaced by the image defined by this property.
Other types of values will follow the settings of the -nexa-edge property.
<Theme-Image> 
This value indicates an image defined in a theme file with the format "theme://images\image name".

The relevant image must be defined in the theme file.
<Web-Image> 
You can specify an image located in the Web with the format "http://path/image name" or "https://path/image name".

You must have a Web-enabled environment.
Remark 
- This property is used to apply an inverted image when the flow direction is changed by the rtl property.

- Since this property can be set only in an XCSS file, you cannot change the value of this property through a script.
 If you do not set the value of this property in an XCSS file, the image defined by the -nexa-edge property will be applied.

- A user-prefix defined in Services of TypeDefinition cannot be used as an image path in a theme or XCSS file.

- A background image applies to the client area of a component.

- Even if background image or color is already set by the background or -nexa-rtl-background-image property, the image that is defined by this will top it.


◆ Restrictions in the web runtime environment

- This property may not function properly in the web browser versions that do not support CSS3.
