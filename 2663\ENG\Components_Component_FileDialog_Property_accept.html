Description 
The attribute to set the format of the file to be applied to the FileDialog.
Syntax 
FileDialog.accept[= strAccept]
Setting Syntax 
this.FileDialog00.accept = ".png, .exe"; 
this.FileDialog00.accept = "image/png, application/*"; 
this.FileDialog00.accept = "audio/*, .png, .exe";
strAccept 
You can set the file types applicable to a FileDialog by specifying extensions or MIME types.

You can specify extensions like ".png", ".exe", and ".png, .exe".
You can specify MIME types like "image/png", "application/*" and "audio".

If you can use commas (,) as dividers, you can define multiple file types by indiscriminately specifying both extensions and MIME types: for example, "audio/*, .png, .exe". 

* In the mobile WRE, you can specify only MIME types.
Remark 
◆ Desktop NRE restrictions

The filter attribute should be set to apply a file format to the FileDialog.


◆ Mobile WRE restriction

- Only "image", "video", and "audio" values can be set with the MIME.
   Extension restriction methods such as “image/png” format are not applicable.

- Only the file where the "image", "video", and "audio" files are loaded in each memory can be accessed when the "image", "video", and "audio" are set with the MIME.
The file format set is not applied to files located externally such as Google Drive and NAS.


◆ iOS/iPadOS Default Web Browser restrictions

- The file format set is not applied to the photo storage.
