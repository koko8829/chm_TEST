Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
This property is used to set the format in which the bound data are displayed on the screen when the subcell is not in edit mode.
Syntax 
CellSubCellControl.displaytype[= enumDpType]
Setting Syntax 
enumDpType ::= 'normal' | 'none' | 'buttoncontrol' | 'calendarcontrol' | 'checkboxcontrol' | 'combocontrol' | 'combotext' | 'currency' | 'date' | 'decoratetext' | 'editcontrol' | 'imagecontrol' | 'mask' | 'maskeditcontrol' | 'multicombocontrol' | 'number'
 
this.Grid00.setSubCellProperty( "body", 0, 0, "displaytype", "normal" );
"normal" 
A sub-cell shows bound data according to the data's original type in the Dataset.

If the data is the "DATE" type, the cell will show only the values that can be formatted as "yyyy-MM-dd".
If the data is the "TIME" type, the cell will show only the values that can be formatted as "hh:mm:ss".
If the data is the "DATETIME" type, the cell will show only the values that can be formatted as "yyyy-MM-dd hh:mm:ss".
If the data is the number type, a comma will be used as a thousands separator at every three digits.
"none" 
A sub-cell does not show bound data.
"buttoncontrol" 
A sub-cell constitutes a Button control while the bound data is used as the text of the control. 

A pseudo-class status will be applied when you click or press the space key on the sub-cell.
"calendarcontrol" 
A sub-cell constitutes a Calendar control while the bound data is displayed in the control according to the date format. 

The date/time format follows the value of the calendardateformat property of the cell.
"checkboxcontrol" 
A sub-cell constitutes a CheckBox control while the bound data determines whether the checkbox is checked. 

If the bound data is "true", the relevant checkbox will be checked.
"combocontrol" 
A sub-cell constitutes a Combo control while the control's data items will be displayed if they match the bound data items. 

To make the Combo control work, you must set the combodataset, combocodecol and combodatacol properties.
Among the values of a combodataset's column specified by the combodatacol property will be displayed if they match the values of the combocodecol property.
"combotext" 
A Combo's data items will be displayed in the form of text if they match the data items bound to a sub-cell. 

To display text properly, you must set the combodataset, combocodecol and combodatacol properties.
Values of a combodataset's column specified by the combodatacol property will be displayed if they match the values of the combocodecol property.
"currency" 
The data bound to a sub-cell is displayed according to the denomination rules. 

The denomination rules here are defined by the system.
"date" 
The data bound to a sub-cell is displayed in the form of text according to the date format. 

The date/time format follows the value of the calendardateformat property of the cell.
"decoratetext" 
The data bound to a sub-cell is displayed in the form of text.

If the data contains line feed characters, the line will be broken.
If the data contains decoration tags, the text will be given the relevant decoration effects.
Refer to the Remark section to learn the available decoration tags.

*The decoration effect is not applied when exporting excel because the set tag is in HTML format.
  Depending on the export setting, the tag is exported as text or the text with the tag removed is exported.
"editcontrol" 
A sub-cell constitutes an Edit control while the bound data is used as the text of the control.
"imagecontrol" 
The data bound to a sub-cell is displayed as an image.

Bound data must be either a URL that indicates the location of image or a Binary Large OBject (BLOB) value.
Supported image formats are "BMP","PNG","GIF","JPEG","SVG". As for "AniGIF", only the first frame is displayed while the animation does not work.
"mask" 
The data bound to a sub-cell is displayed in the form of text formatted as defined by the maskeditformat property.

You can define the format of the mask entered to a sub-cell with the maskeditformat property while the maskedittype property is used to determine the form of an input.
"maskeditcontrol" 
A sub-cell constitutes a MaskEdit control while the bound data is displayed in the form of text formatted as defined by the maskeditformat property.

You can define the format of the mask entered to a sub-cell with the maskeditformat property while the maskedittype property is used to determine the form of an input.
"multicombocontrol" 
The Cell area is displayed in the MultiCombo control type, and the data value of the MultiCombo item that matches the bound data is displayed.

The multicombodataset, multicombocodecol, and multicombodatacol property values must be set for the MultiCombo control to work.
Among the multicombocodecol values in the multicombodataset, the multicombodatacol value corresponding to the value that matches the bound data is displayed on the screen.
"number" 
Data bound to the SubCell will be displayed as numbers.

A comma (,"") will be displayed every three digits.
"progressbarcontrol" 
The SubCell area will be displayed in the form of a ProgressBar control, and bound data will be displayed as numbers and blocks in the control.

Bound data will be displayed as a number in the center of the ProgressBar control.
The blocks in the ProgressBar control can display values ranging from 0 to 100.
"radioitemcontrol" 
The SubCell area will be displayed in radioitem form.

If the data bound to the text property value matches the radioitemcodevalue property value, it will be shown to be selected. If the values are different, it will be unselected.
"text" 
Data bound to SubCell will be displayed as text.

If there is a newline character in the bound data, a line break will be inserted.
"textareacontrol" 
The SubCell area will be displayed in the form of TextArea control, and the bound data will be displayed as text.

If there is a newline character in the bound data, a line break will be inserted.
"treeitemcontrol" 
The SubCell area will be displayed in the form of a Tree control, and the bound data will be displayed as text in the control.

For the Tree control to work, you must set the treelevel and treestate property values for the SubCell.
Remark 
- If the displaytype property value is not set, it will be set as "normal."

- This property is commonly applied to the Head, Body, and Summary bands of the Grid.

- If the bound data is in the "BLOB" format, the Suppress function will not be supported.

- When the control is displayed in an area without data due to the fillareatype property value of the SubCell, the "disable" status will be applied.

- A SubCell merged into a form that has separate subcell areas and properties may not be displayed properly when the value is set as "control" format. 
    To merge subcell areas and properties separately, use the "Merge Cells (Having child cells)" menu in the Grid Contents Editor.

- It is in tag format that can be applied when the displaytype property value is "decoratetext."
  If special characters such as "<" and ">" in tag format are used and English alphabetic characters other than the designated tag format are used, the input value will be deleted.
   For example, when entering in the format "<a href=''>," the value will be deleted for processing.
