copyRow

Description 
This method replaces a certain row of the current Dataset with a certain row of the specified Dataset.
Syntax 
Dataset.copyRow( nToRow, objDataset, nFromRow [,strColInfo] )
Parameters
Parameters Type Description
nToRow Number 
This is the index number of a row in the current Dataset. The row will be replaced with new data.
objDataset Object 
This is a Dataset object that contains the data to copy.
nFromRow Number 
This is the index number of a row of the Dataset defined by the objDataset parameter. The row will be copied.
strColInfo String 
This is a string representing the conditions for matching columns. An example of the format is "ToColumnID=FromColumnID,ToColumnID1=FromColumnID1".

If you do not set this parameter, only the columns whose IDs are the same will be matched.
Sample Call:
var bSucc;
bSucc = this.Dataset00.copyRow(0,this.Dataset01,1);
bSucc = this.Dataset00.copyRow(0,this.Dataset01,1,"name=sawon_name,sabun=id_no");
Return
Type Description
Boolean 
If this method is successfully executed, it will return "true".
Otherwise, it will return "false".
Remark 
- If you want to perform a copy operation by matching two columns of different IDs, you can designate a column to copy. 
 If you set the parameter strCollinfot, only the designated columns will be copied.
 If no column is designated, all the columns of the same IDs will copied.

- If this method succeeds, the oncolumnchanged event will occur.
At this time, only row-related properties are valid among the properties of the DSColChangeEventInfo object, which is delivered to the oncolumnchanged event. 
 
- If the updatecontrol property is set to "true", the type of the replaced row will become "ROWTYPE_UPDATE". If the property is set to "false", the type of the row will become "ROWTYPE_NORMAL".

- This method changes a row of a specified position. Therefore, if you want to copy a row and then add the copied row, you need to call the addRow() method first to create a new row before calling this method.

- If you specify a NaN value like "undefined" or "null" as a value of the nToRow or nFromRow parameter, note that the values will turn to 0 in compliance with ECMA's conversion rule.

- The onvaluechanged event will occur if the currently-selected row is defined by the nToRow parameter.

- With the datatyperule property set to "2.0", the new data validation rule is applied.
  In this case, you should be careful because data can be treated as invalid depending on the type of the column.
