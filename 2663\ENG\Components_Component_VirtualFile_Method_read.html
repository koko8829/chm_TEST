Description 
The method to read the content of a file opened in VirtualFile
Syntax 
 VirtualFile.read([nLength [, strCharset]])
Parameters
Parameters Type Description
nLength Number 
The length of contents to read from the file, specified in byte.

If omitted, the entire fill will be read.
strCharset String 
It specifies a character set applied to contents to read

This parameter is supported only by the Windows NRE. In mobile devices, the UTF-8 character set will be applied.
Sample Call:
var varValue;
varValue = this.VirtualFile00.read(1000);
varValue = this.VirtualFile00.read(1000, "utf-8");
Return
Type Description
Boolean 
"False" is returned if a character not allowed in the parameter is used.

"True" is returned for all other normal cases.
Remark 
- An onsuccess event occurs when file reading is successful.
   - An onerror event occurs when failing to read a file.

- The content of the file read is delivered as the attribute value of the binary data or texdata of the “e” object (VirtualFileEventInfo) at the onsuccess event.

- In case of a text file, if it includes text set information (BOM:Byte Order Mark) at the beginning, it reads the file as long as nLength excluding the relevant part.

- The location of the file pointer moves after reading the content of the file.

-Due to the limited memory size that can be allocated from the VirtualFile object at one time, files larger than 2 GB cannot be processed when the method is executed.
