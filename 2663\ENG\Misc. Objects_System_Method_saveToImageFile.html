Misc. Objects > System > Method > saveToImageFile
saveToImageFile
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
The method to output the image of the selected component as a file.
Syntax 
system.saveToImageFile(objComponent, strFileName, strImageType[,strOptions [,bWholeForm]])
Parameters
Parameters Type Description
objComponent Object 
This is an object representing a component whose image will be printed.
strFileName String 
This is the name of an image file that will be printed.
strImageType String 
Set the image file format for output.
"BMP", "BMP_MONO", "JPG", "TIF", "TIF_24", "TIF_MONO", "GIF", "PNG" formats can be set.

Only "JPG" and "PNG" formats are supported in the Android NRE.
strOptions String 
This is a string that represents the options for printing an image file, such as a compression rate, compression type and DPI resolution.

You can set a compression rate only when the image type is JPG. You can set a rate with the format "Compression:rate".
The compression rate ranges from 0 to 100.
If you do not specify a compression rate when the image type is JPG, the rate will be set at 50 by default.

You can set a compression type only when the image type is TIF. You can set a type with the format "Compression:type".
You can select a compression type among NONE, LZW, CCITT3 and CCITT4.
If you do not specify a compression type when the image type is TIF, the compression type will be set to "LZW" by default.

As for DPI resolution, the formats for setting horizontal and vertical resolutions are "ResolutionX:resolution" and "ResolutionY:resolution"
The resolution is set in the form of an integer.
If you do not specify resolution, the both horizontal and vertical resolutions will be set at 96 by default.

Options are separated by semicolons like "Compress:95;ResolutionX:300".
bWholeForm Boolean 
This parameter is applicable only when the component is a Form.

If you specify "true", the entire Form including the area out of sight will be printed as an image.
If you specify "false", only the area of the Form within sight of the current screen will be printed as an image.

The default value of this parameter is "false".
Sample Call:
var bSucc;
bSucc = system.saveToImageFile(this.Grid00, "test.jpg","PNG");
bSucc = system.saveToImageFile(this.Button00, "test.tif","TIF_MONO","Compress:CCITT3;ResolutionX:300;ResolutionY:300");
bSucc = system.saveToImageFile(this, "test.png","PNG", "", true);
bSucc = system.saveToImageFile(this.Div00.form, "test.jpg","JPG", "", true);
Return
Type Description
Boolean 
"True" is returned when outputting the image is successful.

"False" is returned when failing to output the image.
Remark 
- The value that can be set in the strFileName differs according to the security policy of the OS.

- The image format should be “PNG” to keep the alpha channel value to save the translucent value.
   However, some deformation may occur on the image if the background color is not “black” due to the characteristics when processing the alpha channel value of the “PNG” file.

- The overall characteristic by image format is as follows although there are differences according to the feature of the component.
   Capacity: BMP > TIF > PNG > JPG(100) > GIF > JPG(0)
   Quality: (BMP = PNG) > TIF > JPG(100) > GIF > JPG(0)

- The ColorDepth is saved as 1bit monochrome unconditionally if the image compression method is “"CCITT3" or "CCITT4”.

- The "BMP_MONO" and "TIF_MONO" image formats are saved as 1bit black and white monochrome images.

- The "TIF_24" image format is saved as a 24bit RGB image.

- The application sandbox policy is applied when saving files.
- The image file storage locations for each operating system are as follows.
  - Windows NRE: Documents
  - macOS NRE: Documents
  - Android NRE:
     - targetSdkVersion 29 or lower: /storage/emulated/0/
     - targetSdkVersion 30 or higher: /storage/emulated/0/Pictures/
