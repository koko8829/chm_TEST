Description 
This property is used to set the border of a DateFieldBoxControl.
Syntax 
DateFieldBoxControl.-nexa-border
Setting Syntax 
-nexa-border ::= <line-width> <line-style> <color> [,<line-width> <line-style> <color>]{3}

<line-width> ::= 'thin' | 'medium' | 'thick' | <nWidth> 'px'
<line-style> ::= 'none' | 'hidden' | 'dotted' | 'dashed' | 'solid' | 'double' | 'groove' | 'ridge' | 'inset' | 'outset'
<color> ::= <NamedColor> | <NumericColor>

One value: the value will apply to all the sides (top, right, bottom and left).
Two values: the two values will apply to the top-bottom and right-left sides consecutively. 
Three values: the three values will apply to the top, right-left and bottom sides consecutively.
Four values: the four values will apply to the top, right, bottom and left sides consecutively.

* The case of entering two, three or four arguments will not work if the border-radius property is set.
 
* XCSS
-nexa-border : 1px solid #999999;
-nexa-border : 1px solid #999999, 1px solid #999999, 1px solid #999999, 1px solid #999999;
<line-width> 
Thickness of a border line, specified in the form of a string

"thin" for 1 pixel width
"medium" for 3 pixel width
"thick" for 5 pixel width

If you specify <nWidth>, this parameter will be overriden by <nWidth>, which specifies the width in pixel.
<nWidth> 
Thickness of a border line, specified in pixel
You cannot omit the unit "px".
<line-style> 
Style of a border line

none: specifies no border
hidden: specifies no border
dotted: specifies a dotted border
dashed: specifies a dashed border
solid: specifies a solid border
double: specifies a double border
groove: specifies a 3D grooved border
ridge: specifies a 3D ridged border
inset: specifies a 3D inset border, which makes the inner component appear to cave in
outset: specifies a 3D outset border, which makes the inner component appear to protrude
<color> 
Sets the color of a border by specifying a color name or color code
<NamedColor> 
You can specify a predefined color name in the form of a string like "red" and "blue".
<NumericColor> 
This is a code that indicates the color to display.

The code is a hexadecimal values that begins with "#" and comprises the following 6-digit value (eg. #FF00FF).
You can specify a color code with the rgb() method whose parameters are red, green and blue values. (eg. rgb(255,0,255))
You can specify a color code with the rgba() method whose parameters are red, green, blue and alpha values. (eg. rgba(255,0,255,0.3))
You can specify a color code with the hsl() method whose parameters are hue, saturation and lightness values. (eg. hsl(120,50%,50%))
You can specify a color code with the hsla() method whose parameters are hue, saturation, lightness and alpha values. (eg. hsla(120,50%,50%,0.8))
Remark 
- If you do not set this property, the value will be "undefined" while the property will behave as if its value is "medium none #999999".

- You cannot set two different colors for two lines when the <line-style> parameter is set to "double".
   The color defined by the <color> parameter will apply to the both lines.

◆ Restrictions in the Nexacro Runtime Environment

- The following describes the priority order applied when colors or widths differ between top, right, bottom and left sides: top > right > bottom > left.

- It does not support "double," "groove," "ridge," "inset" and "outset" as a value of <line-style>.

- It does not support "thin," "medium," and "thick" as a value of <line-width>.


◆ Restrictions in the Web Runtime Environment

- Each browser may have different Z-order applied when colors or widths differ between top, right, bottom and left sides.

- The following rules describes the case where colors or widths differ between top, right, bottom and left sides while rounded corners are applied by the border-radius property: the color of the top line will prevail while each line will maintain their own widths.

- In some browser, the border colors and widths of each side can remain different from each other even if the border-radius is set.
