Description 
The attribute to set the type of text allowed to be entered in the TextArea.
Syntax 
TextArea.inputtype[= strInputtype]
Setting Syntax 
 strInputtype ::= 'normal' | <type>

<type> ::= <inputtype> | <inputtype> ',' <type>
<inputtype> ::= 'alpha' | 'english' | 'digit' | 'number' | 'numberandenglish' | 'comma' | 'dot' | 'sign' | 'space' | 'symbol' | 'half'
 
this.TextArea00.inputtype = "english";                       // Only English characters are allowed 
this.TextArea00.inputtype = "english,number";            // Only English characters and numbers are allowed 
this.TextArea00.inputtype = "english";
"normal" 
It permits all types of characters.
"alpha" 
You can enter lower-case letters (a-z) and upper-case letters (A-Z).
"english" 
You can enter lower-case letters (a-z) and upper-case letters (A-Z).
"digit" 
You can enter numbers (0-9).
"number" 
You can enter numbers (0-9), periods (.), commas (,) and minuses (-).
"numberandenglish" 
You can enter lower-case letters (a-z), upper-case letters (A-Z), numbers (0-9), periods (.), commas (,) and minuses (-).

Setting this value is the same as setting "number" and "english" together.
"comma" 
You can enter commas (,).
"dot" 
You can enter periods (.).
"sign" 
You can enter pluses (+) and minuses (-).
"space" 
You can enter white spaces (" ") and tab characters.
"symbol" 
You can enter the following symbols: !,",#,$,%,@,&,',*,/,\,:,;,^,`,{,},[,],(,),|,~,=,_,<,>,?

You can enter all symbols except for the ones defined by the "comma", "dot", "sign" and "space" values.
"half" 
You can enter half-width characters.
"full" 
You can enter full-width characters.
Remark 
- It is applied as “normal” if the inputtype attribute value is not set.
  When setting the inputtype attribute value, the “normal” value is ignored when setting another value together with “normal”.
  For example, if set the attribute value as “normal, english”, it is applied as “english”.

- The inputtype attribute is applied only when directly input by the user.
  It is not applied when the value attribute value is set with a script.

- It only allows the input of characters belonging to the set type value, and characters of the type value not set cannot be input.

- Only the half-width characters can be input if the “full” value is not included in the inputtype attribute value.

- The attribute to limit the input is applied in the order of inputmode > inputfilter > inputtype.


◆ Mobile restrictions

-If the inputtype attribute is set to allow only number to be input such as "digit", "number", "tel", "dot",
   the tel type keypad is applied as default in the Android, and the number type keypad in the iOS/iPadOS.
   However, in iOS/IPadOS, text type keyboard is applied as default if text other than number is saved in the value.

-If the inputtype attribute is set to allow characters to be input such as "numberandenglish", "english”,
   text type keypad is applied as default in both Android and iOS/iPadOS.


◆ Chrome, Edge restrictions

- If the input value is input with a combination character (ENean, etc.) while the area is selected when the inputtype attribute value is “digit”, the selected area is released and the caret is placed at the very last.
