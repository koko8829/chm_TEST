Description 
DateField is a component for receiving date and time input.
Remark 
- Entering values directly into the editing area is only supported in the desktop environment.
   In the mobile environment, a DateTimePicker is displayed when the edit area is touched, and the value selected in the DateTimePicker is entered.


◆ Action when selecting edit area

- When you right-click in the editing area, the default units are selected as "Year", "Month", "Day", "Hour", "Minute", and "Second".
   For the selected unit, you can enter the value directly with the keyboard or change the value with the up and down arrow keys.

- You can also select an area based on mask characters in the form of "Year, Month, Day", "Hour, Minute, Second", "Year, Month, Day, Hour, Minute, Second".
   Plain strings, excluding mask characters, are selected if they are contained in the middle of the selection, but are not selected if they are at the beginning or end of the selection.
   For example, if the format property value is "<yyyy-M-d>H:m:s", you cannot select values containing "<" and ">".

- You can paste the value corresponding to the selected mask character.
   For example, if the format property value is "yyyy-M-d", select the "Year, Month, Day" area and "19940530". When you paste values such as "1994/05/30" and "1994/5/30", the values are entered according to the format property value.


◆ Last date correction of existing value

  - When the year or month is changed, if the date of the existing value is greater than the last date of the changed year and month, it is corrected to the last date of the changed year and month.


◆ Pop-up calendar closes after opening

- If the DateField dropbutton image area and the pop-up calendar overlap, the pop-up calendar may close immediately after opening.
   This is an issue where the clickable positions overlap. You must change the settings so that the dropbutton image area and the pop-up calendar do not overlap.

Constructor 
DateField.init( strName, vLeft, vTop , vWidth, vHeight [, vRight, vBottom, [vMinWidth, [vMaxWidth, [vMinHeight, [vMaxHeight]]]]] )
Parameters Type Description
strName String 
It sets the ID of a DateField in the form of a string.
vLeft String, Number 
It specifies the value of the left property of a DateField in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the right property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vTop String, Number 
It specifies the value of the top property of a DateField in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the bottom property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vWidth String, Number 
It specifies the value of the width property of a DateField in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vHeight String, Number 
It specifies the value of the height property of a DateField in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vRight String, Number 
It specifies the value of the right property of a DateField in pixel or percent (%).

This parameter will be ignored if both the vLeft and vWidth parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the left property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vBottom String, Number 
It specifies the value of the bottom property of a DateField in pixel or percent (%).

This parameter will be ignored if both the vTop and vHeight parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the top property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vMinWidth String, Number 
It specifies a DateField's minimum width that can be displayed in the Form.
vMaxWidth String, Number 
It specifies a DateField's maximum width that can be displayed in the Form.

If this value is smaller than vMinWidth, it will be replaced by the value of vMinWidth.
vMinHeight String, Number 
It specifies a DateField's minimum height that can be displayed in the Form.
vMaxHeight String, Number 
It specifies a DateField's maximum height that can be displayed in the Form.

If this value is smaller than vMinHeight, it will be replaced by the value of vMinHeight.

objDateField.init("DateField", null, null, 300, "400px", "80%", 300);
or
var objDateField = new DateField("DateField", 30, 120, 196, 46, null, null);
var objDateField = new DateField("DateField", null, null, 300, "400px", "80%", 300);
 
// Add Object to Parent Form  
this.addChild("DateField", objDateField); 
 
// Insert Object to Parent Form  
this.insertChild(1, "DateField", objDateField); 
 
// Remove Object form Parent Form  
this.removeChild("DateField"); 
 
// Show Object

// Destroy Object  
objDateField.destroy();
