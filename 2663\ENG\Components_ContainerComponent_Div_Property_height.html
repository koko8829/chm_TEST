Property Type

Description 
This property is used to set the height for displaying the Div.
Syntax 
Div.height[= nHeight]
Setting Syntax 
nHeight ::= [<strCompID> ':'] <nVal> ['px' | '%' | 'em' | 'rem']
 
this.Div00.height = 10; 
this.Div00.height = "10px"; 
this.Div00.height = 10%"; 
this.Div00.height = Button00:10%";
<strCompID> 
It specifies the ID of a component, which serves as a base for determining the height of a Div relatively.

The Div and its parent must have the same base component.
Moreover, the base component must be created earlier than the Div.

If you do not specify a value or specify an invalid value, the height will be determined based on the parent.
<nVal> 
Set the height of the Div as a number in the absolute length unit ("px"), ratio unit ("%"), and relative length unit ("em," "rem").

* When the <strCompID> value is omitted:
- "px": set the height of the Div
- "%": The height of the Div is determined based on the height property value of the parent
- "em": The height of the Div is determined based on the font size property value of the component
- "rem": The height of the Div is determined based on the font size property value of the main frame

* When the <strCompID> value is set:
- "px": The <strCellID> value is ignored and the height of the Xcomponent is determined by the set value
- "%": The height of the Div is determined based on the height property value of <strCompID>
When setting the <strCellID> value, "em" and "rem" units are not supported.
Remark 
- If the unit is omitted in the height value, the pixel value will be applied.

- If the height value includes a reference component or unit character, it must be set in string format. 

◆ Things to keep in mind when setting reference components

- If you set the property value to refer to a reference component (<strCompID>), be sure to the following matters in mind.

    

  • When changing the reference component value (<strCompID>) set in a property value with a script, if it is not a valid component value (incorrect component ID or component with different parent or stepindex), the reference component value will be ignored and the coordinate value (<nVal>) based on the parent will be applied.
    • 

  • If the size or position property value of the reference component is changed with a script (including changes due to the execution of a method that affects the property value), the changed property value will not be immediately reflected in the component that references the reference component.
For example, if the left property value of the Button01 component is "Button00:10px," even if the right property value of the Button00 component is changed by a script, the position of the Button01 component will not change.
To reflect a change in the size or position property value of the reference component in the component that references the reference component, you must execute the resetScroll method of the Form object in the script after changing the size or position property value of the reference component.
