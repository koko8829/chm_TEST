box-shadow

Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
This property is used to set the shadow effect on DateFieldBoxControl.
Syntax 
DateFieldBoxControl.box-shadow
Setting Syntax 
box-shadow ::= 'none' | [<inset>] <offset> [<blur>'px'] [<spread>'px'] [<color>]

<offset> ::= <horizontal>'px' <vertical>'px'
<color> ::= <NamedColor> | <NumericColor>
 
* XCSS
box-shadow : 10px 10px 10px #888888;
box-shadow : inset 10px 10px 10px 5px #888888
<inset> 
The shadow effect will be displayed inside the component.
If omitted, it will operate as an outset.
NRE is not supported.
<offset> 
Set where the shadow effect should be displayed. 

Set the position, where the shadow effect should be displayed based on the upper left position of the Xcomponent, as a horizontal/vertical pixel value.
<horizontal> 
Set the horizontal position where the shadow effect should be displayed.

If a negative number is set, the shadow effect will appear on the left side of the component.
It should be entered as pixels and the "px" unit cannot be omitted.
<vertical> 
Set the vertical position where the shadow effect should be displayed. 

If a negative number is set, the shadow effect will appear at the top of the component.
It should be entered as pixels and the "px" unit cannot be omitted.
<blur> 
Set the blur for the shadow effect.
If a negative number is set, the blur will not be applied.
It should be entered as pixels and the "px" unit cannot be omitted.
<spread> 
This will enlarge or shrink the shadow.
Setting a positive number will enlarge the shadow, and setting a negative number will shrink the shadow.
NRE is not supported.
<color> 
The color of the shadow effect can be set using a color name or color code.

If a value is not entered, it may operate differently or malfunction depending on the browser.
<NamedColor> 
Set as a predefined color name such as "red" and "blue" as a string.
<NumericColor> 
The following are color codes: 

You can set the color code as a 6-digit hexadecimal value starting with the "#" symbol. (ex-"#FF00FF")
You can set the color code in the "rgb()" method, which uses red, green, and blue values as arguments (ex-"rgb(255,0,255)"). 
You can set the color code in the "rgba()" method, which uses red, green, blue, and alpha values as arguments (ex-"rgba(255,0,255,0.3)").
You can set the color code in the "hsl()" method, which uses hue, saturation, and lightness values as arguments (ex-"hsl(120,50%,50%)").
You can set the color code in the "hsla()" method, which uses hue, saturation, lightness, and alpha values as arguments (ex-"hsla(120,50%,50%,0.8)").
Remark 
- If the box-shadow property value is not set, it will be set as undefined and operates as "none."

- If <blur> is set as a large value, system resource requirements will increase, thereby lowering performance. 

- <inset> and <spread> values do not support NRE.
   Even in the Nexacro Studio design window, you will not be able to check the changed design when setting property values.
