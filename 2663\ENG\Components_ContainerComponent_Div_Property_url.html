Description 
This property is used to connect a Form to a Div component.
Syntax 
Div.url[= strUrl]
Setting Syntax 
strUrl ::= <absolute-url> | <relative-url>
 
this.Div00.url = "http://localhost:8080/Prj/Base/temp00.xfdl.js";     // Absolute path

this.Div00.url = "../Base/temp00.xfdl";                                        // Relative path
this.Div00.url = "Base::temp00.xfdl";                                           // Path to the Services in the TypeDefinition
<absolute-url> 
It specifies a generated xfdl file on the Web with the format "http://path/Form name" or "https://path/Form name". 

You must be able to access the Web, and the Form to specify must be generated.
The use of this property is limited in the cross-domain situations.
<relative-url> 
It specifies a path by using a prefix defined in Services in TypeDefinition.
This is a relative path based on the location of a current Form.
Remark 
- This property is used to connect an existing Form to a Div.

- The size of a Div will remain unchanged regardless of the size of the connected Form.

- The application may malfunction if you attempt to change the url property inside the connected Form and if the async property is set to "false".
 If the async property is set to"false", a new Form will be loaded as soon as you change the url property. Then, the existing form will be deleted before the execution of the relevant script is completed, which will result in terminating the application abruptly.

- If you changes the url property inside the connected Form, the remaining portion of the script after the point of changing the URL will not be executed.

- This property cannot connect a Form to which the Div belongs or Forms at higher levels.
