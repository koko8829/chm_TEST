Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
The attribute to set the horizontal alignment method of the text or content displayed in the cell.
Syntax 
GridCellControl.textAlign[= strtextalign]
Setting Syntax 
strtextalign ::= 'left' | 'center' | 'right'
 
* XCSS
-nexa-text-align : center;

* Script ( normal property )
this.Grid00.setCellProperty( "body", 0, "textAlign", "center" );
"left" 
Text is or contents are aligned with the left side of a Grid cell.
"center" 
Text or contents are aligned with the center of a Grid cell.
"right" 
Text or contents are aligned with the right side of a Grid cell.
Remark 
- “left” is displayed on the Grid content Editor when the -nexa-text-align attribute value is not set and there is no value defined in the XCSS.
   It is operated in the following ways when there is no value defined in the XCSS nor in the XCSS of the control determined by the displaytype attribute value.
   > It changes according to the column type of the bound dataset if the displaytype attribute value is “normal”.
   > When the displaytype attribute value is “mask”, it is operated as “left” if the maskedittype attribute value is “string”, and “right” if “number”.
   > It is operated as “left” if the displaytype attribute value is "maskeditcontrol, progressbarcontrol, textareacontrol, editcontrol, combocontrol, calendarcontrol".
   > It is operated as “center” if the displaytype attribute value is "combotext, date, text, decoratetext, buttoncontrol, treeitemcontrol".
   > It is operated as “right” if the displaytype attribute value is "number, currency".
   > The Control itself is located in the center of the Cell if the displaytype attribute value is "checkboxcontrol, imagecontrol, radioitemcontrol".

- The -nexa-text-align attribute value is applied to the location where the Control is marked if “control” is included in the displaytype attribute value.
   The text alignment displayed inside the Control changes according to the value defined in the Control.

- It changes according to the column type of the bound dataset if the displaytype attribute value is “normal”.
  > It is operated as “left” if the column type is "STRING, BLOB".
  > It is operated as “right” if the column type is "INT, FLOAT, BIGDECIMAL".
  > It is operated as “center” if the column type is"DATE, TIME, DATETIME".

- "textAlign" should be used for the attribute name when accessing with a script.

- If the text length displayed is bigger than the width of the cell, it is operated as “left” even if it is set as “right”.


◆ web runtime environment restrictions

- The tilted part of the last text may be cut when displayed if the font attribute is set as “italic” when the -nexa-text-align attribute value is “right”.
   This problem occurs when applying tilting after handling the text area based on regular characters in the web browser.
