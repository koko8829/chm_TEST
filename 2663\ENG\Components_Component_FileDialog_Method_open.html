Description 
This is a method of displaying a FileDialog.
Syntax 
FileDialog.open( strTitle, constOpenMode [, strInitialPath [, strFileName [, nWidth [, nHeight ]]]] )
Parameters
Parameters Type Description
strTitle String 
This is a string that is displayed on the title bar of a FileDialog.

* This property does not apply to the macOS NRE and web runtime environment.
constOpenMode Constant 
This parameter defines the type of the FileDialog to open.

If you specify "FileDialog.LOAD" or 1, this method will open a FileDialog that can read a single file.
If you specify "FileDialog.SAVE" or 2, this method will open a FileDialog that can save a single file.
If you specify "FileDialog.MULTILOAD" or 3, this method will open a FileDialog that can read multiple files.
If you specify "FileDialog.SELFOLDER" or 4, this method will open a FileDialog that can select a folder.

If you specify an invalid value, the value "FileDialog.LOAD" will be applied.

* In the desktop WRE, you can apply only the "FileDialog.LOAD" and "FileDialog.MULTILOAD" options.
* In the mobile environment, you can only apply the "FileDialog.MULTILOAD" option to the iOS/iPadOS WRE or iOS/iPadOS NRE(iOS/iPadOS 14 or later).
strInitialPath String 
Set the default folder to be selected when the FileDialog is displayed.
This is set using an absolute path or Alias.

* The set value is not applied in the mobile NRE.
* The set value is not applied in the WRE.
strFileName String 
This parameter defines the name of a file that will be designated by default when a FileDialog is displayed.

When the constOpenMode parameter is set to "FileDialog.SELFOLDER", this parameter will be ignored.

* This parameter is not applied in web runtime environment.
nWidth Number 
This is a number representing the width of the FileDialog, expressed in pixel.

If you specify a number smaller than 648 or do not set this parameter, the value of this parameter will be either 648 or the last-displayed width.
If you specify a value bigger than the width of the monitor, the monitor's width will be set as a value of this parameter.

* This parameter is applicable only to the Windows NRE.
nHeight Number 
This is a number representing the height of the FileDialog, expressed in pixel.

If you specify a number smaller than 356 or do not set this parameter, the value of this parameter will be either 356 or the last-displayed height.
If you specify a value bigger than the height of the monitor, the monitor's height will be set as a value of this parameter.

* This parameter is applicable only to the Windows NRE.
Sample Call:
var varRet;
varRet = this.FileDialog00.open( "FileOpen", FileDialog.LOAD );
varRet = this.FileDialog00.open( "FileSave", FileDialog.SAVE );
varRet = this.FileDialog00.open( "FileSave", FileDialog.SAVE, "%UserApp%" );
varRet = this.FileDialog00.open( "FileSave", FileDialog.SAVE, "%UserApp%", "abc.txt" );
varRet = this.FileDialog00.open( "FileSave", FileDialog.SAVE, "%UserApp%", "abc.txt", 800, 600 );
Return
Type Description
Boolean 
If the method execution is successful, true will be returned. 
If the method execution fails or the parameter value is set incorrectly, false will be returned.
Remark 
- The following script will be executed immediately after the open() method is executed, and an onclose event will occur when the FileDialog is closed after selecting a file.

- The file format to be applied to FileDialog can be set with the filter or accept property depending on the environment.

- If nWidth and nHeight are set, FileDialog will be displayed in the center of the monitor.

- This is the actual Alias path that can be used in strInitialPath.
