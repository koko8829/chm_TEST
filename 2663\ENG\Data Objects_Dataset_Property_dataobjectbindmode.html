Description 
This property is used to set the communication method with the bound DataObject object.
Syntax 
Dataset.dataobjectbindmode[= strDataobjectbindmode]
Setting Syntax 
strDataobjectbindmode ::= "oneway" | "twoway"
 
this.Dataset00.dataobjectbindmode = "twoway";
strDataobjectbindmode 
"oneway": One-way communication. It only supports the action of loading data on a DataObject object.
"twoway": Two-way communication. According to the Dataset object setting, when changing data, the change information is delivered to the DataObject object.
Remark 
- The default value of this property is "oneway".

- When the dataobjectbindmode property value is set to "twoway" and the Dataset object data is changed, the event of the bound DataObject occurs as shown below:

 * Change the row, such as adding, deleting, or changing the position of data
  The ondatachanged event occurs.
  - updatecontrol = true
      An event occurs when the applyChange method is executed.
  - updatecontrol = false
      An event will occur when the Row is changed.
      When executing a method that changes multiple rows, an event occurs more than twice.
             For example, when the exchangeRow method is executed, the method is executed once, but the ondatachanged event occurs twice because the data of the two rows is changed and the positions of the two rows is changed.

 * Change the Column value of a specific Row
    The event that occurs depends on the value of the updatecontrol property.
    - updatecontrol = true
        When the applyChange method is executed after changing the column value, the ondatachanged event occurs.
        Even if only a specific column value is changed and the applyChange method is executed, the ondatachanged event occurs instead of the onvaluechanged event because it is treated as a row change.
    - updatecontrol = false
        The onvaluechanged event occurs when the column value is changed.

- Two-way communication is not supported if a filter script is applied when setting the dataobjectpath property. For example, when the dataobjectpath property value is set, the filter script is applied in the format "$.data[?(@.employee_age==31)]".
   If data filtering is required during two-way communication, use the filterstr property.
