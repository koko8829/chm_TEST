Description 
It’s a virtual file object used to read or use files on the local system.
Remark 
- When executing the read and write methods, it may not be possible to process files larger than 2GB due to memory limitations. Also, the file size that can be processed may differ depending on the OS and the equipment used.

◆ web runtime environment restrictions

- The VirtualFile object cannot be used alone in the web runtime environment.
   It can use VirtualFiles that are provided only with the filelist attribute of the FileUpload, FileUpTransfer, 
   virtualfiles attribute of the FileDialogEventInfo, and the filelist attribute of the DragEventInfo, GridDragEventInfo, ListViewDragEventInfo, etc.

- So the open(), close(), and getFileSize() methods and the filename attribute that are displayed to support some web browsers in the web runtime environment
   can only be used when the VirtualFile object is provided with the attributes listed above.
Property
Name Description
async This property is used to determine how a VirtualFile object performs a requested task.
filename This read-only property holds the name of a file loaded onto a VirtualFile.
fullpath This read-only property holds the information on the full path of a file opened in a VirtualFile.
id This property is used to set the identification of a VirtualFile.
name This property is used to specify the name of a VirtualFile.
parent This read-only property holds the parent object of a VirtualFile.
path This read-only property holds the information on the path of a file opened in a VirtualFile.
Method
Name Description
addEvent This method is used to declare a certain event and then add it to a VirtualFile.
addEventHandler This method is used to add a handler to a certain event of a VirtualFile.
clearEventHandler This method is used to delete all the event handlers registered in a certain event.
close This method is used to close a file opened by a VirtualFile.
copy This is a method of copying a specific file from VirtualFile.
createDirectory This method is used to create a folder in a VirtualFile.
deleteDirectory This method is used to remove a folder in a VirtualFile.
findEventHandler This method finds a handler in a certain event of a VirtualFile and returns the index of the handler.
getEventHandler This method returns a specific handler from a certain event of a VirtualFile object by specifying the index number of the handler.
getFileList This is a method of returning the files or subfolders in a specific location as a list.
getFileSize This method is used to obtain the size of a file opened in a VirtualFile.
insertEventHandler This method is used to add a handler to a certain event of a VirtualFile at a specified position.
isExist This is a method of returning whether a specified file or folder exists.
open This method is used to open a file loaded onto a VirtualFile in a specified mode.
read The method to read the content of a file opened in VirtualFile
readLine This method is used to read the contents of a file opened in a VirtualFile up to a certain point.
remove This is a method of deleting the file specified in VirtualFile.
removeEvent This method is used to remove a certain event that is declared in a VirtualFile.
removeEventHandler This method is used to remove a handler in a certain event of a VirtualFile.
rename This is a method of moving the location of a specific file or change the name of the file in VirtualFile.
renameDirectory This method is used to rename a folder in a VirtualFile.
seek This method is used to relocate a file pointer in a file opened in a VirtualFile.
setEventHandler This method is used to change the first handler in a certain event of a VirtualFile.
write The method to save the data of a file opened in VirtualFile.
Event
Name Description
onerror This event occurs when a task requested to a VirtualFile fails.
onsuccess This event occurs when a task requested to a VirtualFile succeeds.
