showModalSync
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method is used to display a dynamically-created ChildFrame in the form of a modal window that works in a synchronous manner.
Syntax 
system.showModalSync( objChildframe [, strID ] , objParentFrame, [{objArguList} [, objOpener]] );
Parameters
Parameters Type Description
objChildframe Object 
This a ChildFrame object that will be displayed as a modal window.
strID String 
This is a string that represents the ID of a ChildFrame that will be opened as a modal window.

If you omit this parameter, the argument will adopt the name property of the ChildFrame.
objParentFrame Object 
This is an object representing the parent frame of the ChildFrame displayed as a modal window.

If you specify "null", the MainFrame will serve as a parent.
objArguList Object 
This a list of variables to add to a ChildFrame that takes the form of a modal window, with the format {variable name:variable value, variable name:variable value}.
objOpener Object 
This a Form object that will be the opener of the ChildFrame displayed as a modal window.

If unspecified, the opener will be the Form connected to the frame designated by the objParentFrame parameter.
If you specify "null" as a value of the objParentFrame parameter, this parameter will be undefined.
Sample Call:
var objFrame = new ChildFrame();
var objParentFrame = this.getOwnerFrame();

objFrame.init( "modalsync00", 0, 0, 500, 500 );
objFrame.formurl = "Base::Test00.xfdl";

var vRtn = objFrame.showModalSync( objParentFrame, {a:'aaa', b:'bbb'}, this );
var vRtn = objFrame.showModalSync( objParentFrame, "modalsync00", null, this );
Return
Type Description
Boolean 
The success or failure of this method is returned when the modal window is closed.
Remark 
* Dynamic creation of ChildFrame 

- This method can apply to a ChildFrame that is created dynamically through script.
   If the ChildFrame is not created dynamically, it may cause an error.

- As for a dynamically-created ChildFrame, the modal window is sized according to the autosize property.
 The autosize property is set to "true" by default.


* Notes about setting parameters 

- You can access a variable set through the objArguList parameter. The access is made from the modal window with the format "this.parent.[variable name]".

- You can call the getPopupFrames() method from the frame set by the objParentFrame parameter and obtain an array as a return value to use it for setting the strID parameter.


* Others 

- The script will be suspended while the method is executed. After the modal window is closed, the script will proceed again.
 If you call the showModal() method, the part of the script after the method calling will proceed regardless of the opening of a modal window.

- If you call this method, a modal ChildFrame will be added to the list of the pop-ups held by the frame defined by the objParentFrame parameter.
 If the modal window is closed, the ChildFrame will be removed from the list automatically with the destroy method executed internally.

- If a modal ChildFrame is opened with its left and top properties having negative values, both will be adjusted to 0.
 If there is not enough room for opening a modal window, the positions of the window's left and top sides will be adjusted so that the title bar can be displayed.

- A modal window is opened based on the top frame in the window area where the frame designated by the objParentFrame parameter belongs.
 If the parent Frame is modeless, the ChildFrame of a popup containing the modeless windown becomes a base.
  If the objParentFrame parameter is set to null, the MainFrame becomes a base.

- Even if you specify "minimize" as a value of the openstatus property of a ChildFrame that will be opened as a modal window, the property will function as "normal".
