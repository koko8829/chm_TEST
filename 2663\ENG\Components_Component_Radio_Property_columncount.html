Description 
The attribute to set the number of items to be displayed in a column in Radio.
Syntax 
Radio.columncount[= nColcount]
Setting Syntax 
this.Radio00.columncount = 5;
nColcount 
This is the number of items displayed in a row. The number must be -1 or larger.

If you specify 0, this property will not be applied and the items will be laid out according to the rowcount property.

This property behaves depending on the the direction, rowcount and fittocontents properties. Please see the below Remark for the details.
Remark 
(1) Actions when setting the default value

- It is applied as 0 if the columncount attribute value is not set.
- The radio item is displayed on one row if the attribute value of the columncount, rowcount, and direction is not set (if default value). 

(2) Action when the attribute value of the columncount and rowcount 1 or higher

- The rowcount attribute value is ignored if the columncount attribute value is 1 or higher and the direction attribute value is set as "horizontal". Also, the rowcount attribute value is ignored if the columncount attribute value is 1 or higher and the rowcount attribute value is set as 0 or -1.
- The columncount attribute value is ignored if the rowcount attribute value is 1 or higher and the direction attribute value is set as "vertical". Also, the columncount attribute value is ignored if the rowcount attribute value is 1 or higher and the columncount attribute value is set as 0 or -1.

(3) Action when the attribute value is set as -1

- If the fittocontent attribute value is "none" when the columncount and rowcount attribute values are set as -1, the Radio item is automatically allocated according to the length of the item text. If the fittocontent attribute value is not "none", the Radio item is displayed on one row.

(4) Maintaining the current status for wrong actions

The 3 actions below are wrong, but it is not to be modified but to keep the current status to secure the actions of the code written by the existing user. These actions may change after an update, so keep it in mind when writing a new code.

(4-1) The radio item is displayed on one column if the attribute value of the columncount and rowcount is not set and, and only the direction attribute value is set as “vertical”. It is recommended to set the rowcount attribute value as 1 to display the radio item on one column.

(4-2) The radio item is automatically allocated according to the length of the item text when the attribute value of both columncount and rowcount is -1 and the fittocontent attribute value is “none”. At this time, the value set as the direction attribute value is ignored and processed as “horizontal”. It is a wrong action to ignore the direction attribute value, it is maintained as the current status without modification to secure the action of the code written by the existing user.

(4-3) It operates as the following if the columncount attribute value is set as 0 and the rowcount attribute value as -1, or the column count attribute value as -1 and the rowcount attribute value as 0. It is recommended to set the rowcount attribute value as 1 to display the radio item on one column, and to set the columncount attribute value as 1 to display the radio item to display the radio item on one row.
It is recommended to set the columncount attribute value as 0 and the rowcount attribute value as -1 to display the radio item on one column.
- If the columncount attribute value is set as -1, the rowcount attribute value as 0, and the direction attribute value as “vertical”, the radio item is displayed on one column. If only the direction attribution value is set has “horizontal”, the radio item is displayed on one row.
