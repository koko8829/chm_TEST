cellcalendarpopuptype
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
This property is used to determine how to display a pop-up calendar (a date picker) when a Grid cell is calendar type.
Syntax 
Grid.cellcalendarpopuptype[= enumType]
Setting Syntax 
enumType ::= 'none' | 'normal' | 'center' | 'system'
 
this.Grid00.cellcalendarpopuptype = "system";
"none" 
It does not display a popup calendar.
"normal" 
It follows the general rules of displaying a pop-up calendar.
"center" 
It displays a pop-up calendar at the center of the application screen.
"system" 
This value works as "normal" in the desktop environment.

In the mobile environment, the cell displays a pop-up calendar provided by the system.

In the mobile environment, the calendardateformat and calendareditformat properties are not applied to the date format. This property behaves in the ways described below.
Since the date format cannot be changed due to system restrictions, you should bear that point in your mind.
  > In the Android WRE or iOS/iPadOS WRE, the date format is obtained from the browser or system.
  > In the Android NRE or iOS/iPadOS NRE, the date format is obtained from the system.
  > In the iOS/iPadOS NRE, the "yyyy.MM.dd" format is applied regardless of the system's date format.

* The user-defined calendardateformat and calendareditformat properties do not change when restrictions related to a date format are implemented.
* The sequence of events is oninput -> canchange -> onchanged when you select another date in a pop-up calendar and then determine the date in Android or when you stop changing a date in the spinner of a pop-up calendar in iOS/iPadOS.
Remark 
- The default value of this property is "undefined", which works as "normal" and "system" in the desktop and mobile environments respectively.

- If you do not set the calendarpopuptype property of a cell, the value of the cellcalendarpopuptype of the Grid will be applied.

- This property is applicable only when the calendartype property is set to "normal". Moreover, this property should be used restrictively in particular environments like the mobile environment.

- If you use multiple monitors, a pop-up calendar will be displayed based on the monitor screen where the Grid is displayed.
