Description 
This property is used to determine how to load a Form onto a Div component when the value of the url property is dynamically changed.
Syntax 
Div.async[= bAsync]
Setting Syntax 
bAsync ::= 'true' | 'false'
 
this.Div00.async = false;
true 
A Form is loaded in an asynchronous manner.

You can access the components inside the Form after the onload event of the Form.
If you attempt to access the components inside the Form before the onload event, an error will occur.

See the Remark section to learn about executing the portion of a script after the point where the url property is changed.
false 
A Form is loaded in a synchronous manner.

You can access the components inside the Form immediately after you change the value of the url property.
The portion of a script after the point where the url property is changed will be executed after the onload event of the Form.
Remark 
- The default value of this property is "true".

- A Form connected to a Div will be loaded in a synchronous manner when the parent Form that includes the Div is loaded for the first time. 

- An application may malfunction if you change the Div's url property while the async property is set to"false".
 With async set to false, changing url leads to the immediate loading of a new Form, which in turn will cause the existing Form to be deleted. This deletion, which occurs before the completion of the script, is the cause of the malfunction.

- With the async property set to "true," the portion of a script after the change in the url property will be executed at different timings according to which cache the Form is loaded from.
 If the Form is loaded from the cache managed by Nexacro, the portion will be executed after the completion of loading the Form.
 If the Form is loaded from the cache managed by the server or browser, the portion will be executed regardless of the Form loading.
