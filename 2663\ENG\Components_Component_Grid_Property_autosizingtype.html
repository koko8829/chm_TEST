Description 
This method determines how to resize the rows and columns of a Grid component automatically so that all the data will be displayed.
Syntax 
Grid.autosizingtype[= enumType]
Setting Syntax 
enumType ::= 'none' | 'col' | 'row' | 'both'
 
this.Grid00.autosizingtype = "both";
"none" 
This value indicates that the widths of columns will not be adjusted automatically, and they will remain as designed.

The heights of rows will not be adjusted automatically either, and they will remain as designed.
"col" 
This value indicates that the widths of each column will be automatically adjusted. The adjustment will be based on the data that is contained in the area defined by the autosizebandtype property.

The heights of rows will not be adjusted automatically, and they will remain as designed.
"row" 
This value indicates that the widths of columns will not be adjusted automatically, and they will remain as designed.

The heights of each row will be automatically adjusted. The adjustment will be based on the data that is contained in the area defined by the autosizebandtype property.
"both" 
This value indicates that the widths of each column will be automatically adjusted. The adjustment will be based on the data that is contained in the area defined by the autosizebandtype property.

The heights of each row will also be automatically adjusted. The adjustment will be based on the data that is contained in the area defined by the autosizebandtype property.
Remark 
- The default value of this property is "none".

- Even if rows or columns are resized by this property, the resizing will apply only to the rows and columns as displayed, not to the formats property.

- When Dataset object is newly loaded or Cell data is changed, Row and Column size are automatically adjusted every time.
  In addition, when the data to be displayed on the Grid component changes due to the change of the keystring property value of the Dataset object, the size of Row and Column is automatically adjusted.

◆ Action when both autofittype and autosizingtype properties are set

- If the autofittype property value is set to "col" and the autosizingtype property value is set to "col" or "both", the autosizingtype property value for column size is ignored.
   With the autofittype property value set to "col"
   If the autosizingtype property value is set to "col", it is treated as "none".
   If the value of the autosizingtype property is set to "both", it is treated as "row".

◆ Auto-adjust Row and Column ranges

- The autosizebandtype property is used to define the range where rows and columns are resized automatically.
  In this case, merged cells will be excluded from the calculation of row and column sizes.
  If you do not set autosizebandtype, this property will automatically resize the rows and columns whose band properties are set to "body".

◆ Adjust each row to a different height

- With this property value set to "both" or "row" and with the extendsizetype property value set to "row", it is adjusted the heights of each row differently.
   Without extendsizetype set, all the rows will be automatically adjusted in the same height.

- If each row is set to be adjusted to a different height, it operates as follows:
  - It calculates and outputs only the row size of the area shown on the screen.
  - It calculates the row size of the area shown on the screen when scrolling, and when all row size calculations are finished, no further calculation processing is performed.
  
- The calculation time according to the scroll operation method is as follows:
  - When the scroll operation is completed after moving the scroll bar using the track button or scrolling by fling
  - When the scroll bar is moved using the track button and remains stopped for a certain period of time
    (The criteria for handling a stopped state may vary depending on the execution environment. If it is stopped between 0.5 and 1 second, it will be processed.)
  - When the scroll operation is handled by a method other than a track button or fling (mouse wheel button, keyboard arrow keys, etc.) and scrolling is in progress

- When the getCellRect or getSubCellRect method is executed, a different value may be returned depending on whether the Row area set as a parameter is calculated. Non-counted area returns uncounted Cell area information.

- When the getRealRowSize or getRealRowFullSize method is executed, the height value is calculated separately from the scroll operation and returned.

◆ Automatic adjustment of specific Rows or Columns only

- If you want to automatically resize certain rows or columns, you should use the autoSizeCol() or autoSizeRow() method.

◆ Precautions when using the mergeCell method

- When a cell merged by the mergeCell() method bears a high level of complexity, the autosizingtype property may not work as set.
   That is, the result may differ depending on the sequence between setting autosizingtype and calling mergeCell(). 
   Such a disparity is an inevitable restriction for improving performance when the complexity of a merged cell is high. In that case, therefore, you are recommended to call mergeCell() first and then set autosizingtype.
