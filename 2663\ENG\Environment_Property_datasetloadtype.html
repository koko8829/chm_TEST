Description 
This property determines how to load data onto a Dataset when you receive an error code.
Syntax 
Environment.datasetloadtype[= enumLoadType]
Setting Syntax 
enumLoadType ::= 'onsuccess' | 'errorallow'
 
var objEnv = nexacro.getEnvironment();
objEnv.datasetloadtype = "errorallow";
"onsuccess" 
Data will not be loaded onto a Dataset if an error code is a negative number.
If an error code is 0 or larger, data will be loaded onto the output Dataset.

If an error code is a negative number, this property will not update the status of the input Dataset as well as the bound components.
If an error code is 0 or larger, the status of the input Dataset will change to "Normal" with the bound components also updated.
"errorallow" 
Data will be loaded onto the output Dataset regardless of an error code value.

If an error code is a negative number, this property will not update the status of the input Dataset as well as the bound components.
If an error code is 0 or larger, the status of the input Dataset will change to "Normal" with the bound components also updated.
Remark 
- The default value of this property is "onsuccess".

- This property will be applied even when you receive a response after the calling of the load() method of a Dataset.

- An output Dataset is processed before an input Dataset during a transaction.
   Accordingly, data will be loaded if the following conditions are met. First, you set the same Dataset for both an input Dataset and an output Dataset. Second, the error code is a negative number with the datasetloadtype property set to "errorallow".
