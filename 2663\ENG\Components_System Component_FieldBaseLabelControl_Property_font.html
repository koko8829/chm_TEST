Property Type

Description 
This property is used to set the font used in FieldBaseLabelControl.
Syntax 
FieldBaseLabelControl.font
Setting Syntax 
strfont ::= [<font-style>] [<font-weight>] <font-size> ['/'<line-height>] <font-family>

<font-style> ::= 'normal' | 'italic'
<font-weight> ::= 'normal' | 'bold' | '100' | '200' | '300' | '400' | '500' | '600' | '
 
* XCSS
font : bold 12pt/30px arial, sans-serif;
<font-style> 
It sets the style of a font.

Nothing applies if you specify the font as "normal".
The text is shown in italics if you specify the font as "italic".
<font-weight> 
It sets the weight of a font.

Nothing applies if you specify the font as "normal."
The text is shown in bold if you specify the font as "bold."

If you specify a number ranging from 100 to 900, the thickness of the font will be as much as specified.
If you want to specify a number, you must predefine a weight that matches the specified number so that the relevant font is ready to provide the relevant weight.
<font-size> 
Font size units can be set in absolute length units and relative length units as shown below:
If you do not specify a unit, the property value will not be applied.
- Absolute length units: pixel("px") and point("pt")
- Relative length units: "em" and "rem"

If the font size unit is set as "em" or "rem," the font size will be calculated as follows:
- "em": The relative size is set based on the font size of the parent component
- "rem": The relative size is set based on the font size of the main frame
<nSize> 
Set as an integer or decimal depending on the font size units.
Negative values cannot be set.
- "px," "pt": integer
- "em," "rem": integer or decimal
<line-height> 
It specifies line spacing if text is displayed in multiple lines.

You cannot specify a negative number.

If you specify "normal", this option will not have a particular value.
If you specify "normal", line spacing will vary according to web browsers.
If you specify "normal", the Nexacro Runtime Environment will multiply the font size by 1.15 (i.e. the value of <nValue> is 1.15).
<nValue> 
It specifies a value that will be multiplied by a font size.

The result will be applied as the line spacing of the font.
<nPixel> 
If specifies line spacing in pixel.

You cannot omit the unit "px".
<nPercent> 
It specifies a percentage that will be multiplied by a font size.

The result will be applied as the line spacing of the font.
You cannot omit the unit "%".
<font-family> 
You can designate a list of fonts to be applied to a FieldBaseLabelControl.

You can designate multiple fonts by dividing them with commas (,).
Fonts will be applied under a fallback system.
If the system does not support the first font, for example, it tries the next font, and so on.
The system will apply its default font if it can support none of the designated fonts.
<font-name> 
It designated a font to apply to a FieldBaseLabelControl by name.
Remark 
- If the font property is not set, it will be set as undefined and it will operate as "12pt/normal Verdana."

- When the font size of the font property is changed, the line spacing may change depending on the value applied to <line-height>.

- When <font-weight> is set as a number, "400" will be applied as normal thickness, and "700" will be applied as bold.
   If the font does not have a font thickness corresponding to a numerical value, "400" or "700," whichever it is closer to, will be displayed.

- For format information supported by environment when using UserFont, refer to "Appendix > Supported UserFont".


◆ WRE Constraints

- If you set "italic" as the font property when the -nexa-text-align property value is "right," the slanted part of the last letter may not be displayed fully.
    This problem occurs when the web browser processes the text area based on regular characters and then applies italics.
