getPopupFrames
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method returns a list of ChildFrames opened in the window that includes the frame designated as an argument. The list is returned in the form of a collection.
Syntax 
nexacro.getPopupFrames( [objFrame] )
Parameters
Parameters Type Description
objFrame Object 
It is an object representing a base frame.

If unspecified, a base frame may vary according to the conditions that are explained in the Remark section.
Sample Call:
var arrPopFrame = nexacro.getPopupFrames() ;
var arrPopFrame = nexacro.getPopupFrames( this.getOwnerFrame() ) ;
Return
Type Description
Array 
It returns a collection of the frames popping up in the window that includes the frame designated as an argument.

If you specify "null" or an invalid value as the value of the objFrame parameter, this method will return "undefined".
Remark 
- Pop-up ChildFrames are managed based on a window.
 Accordingly, even if this method is called from two different frames, this method will return the same value as long as those two frames belong to the same window.
  For example, you open pop-ups separately in Frame A and Frame B, which both belong the same window. In this case, you can obtain the same result whichever you call this method from.
  However, if Frame A and Frame B belong to different windows, this method will return different values depending on which frame is defined by the objFrame parameter.

- If you omit the objFrame parameter, you may obtain an unintended result as this method returns a value based on the window that is activated at the time of the method calling.
  For example, the open() method is called from Frame A to open Frame B that has its own window. Then, the script defined in A is called to execute the getPopupFrames() method in A. However, getPopupFrames() will take B as its base because the currently-activated window includes B.
  After all, it is recommended that objFrame should not be omitted.

- The open() and showModalWindow() methods open a pop-up in a window separate from the parent.
   The showModal() and showModalSync() methods open a pop-up in the same window as the parent.

- If a ChildFrame is opened by the open(), showModal(), showModalSync() or showModalWindow() method, the ChildFrame will be added to the list of the pop-ups of the window that includes a frame defined by the objParentFrame parameter of the relevant method.

- You can use the collection approach to access a returned object.
  That is to say, you can write codes like ReturnObj[0], ReturnObj["Frame0"], ReturnObj.length.
