onusernotify
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
The event occurring when delivering information from the webpage loaded on the WebBrowser toward Nexacro.
Syntax 
onusernotify(obj:nexacro.WebBrowser,e:nexacro.WebUserNotifyEventInfo);
Parameters
Parameters Type Description
obj WebBrowser 
Event generated Object.
e WebUserNotifyEventInfo 
Event Object.
Return
None
Remark 
- Code below should be defined in the web page to generate the onusernotify website.
  The code to be used changes when the event occurs depending on the execution environment.

  1. For WRE, iOS/iPadOS NRE
     if(window.NEXACROWEBBROWSER)
        window.NEXACROWEBBROWSER.on_fire_onusernotify(window.NEXACROWEBBROWSER, userdata)
  2. For Windows NRE
     if (typeof nexacro == "undefined")
        window.document.title = userdata;
  3. For Android NRE, macOS NRE
     if(nexacro)
        nexacro.fireUserNotify(userdata);

- In the Windows NRE, the maximum number of character that can be set for the userdata is 259.
  (window.document.title attribute is affected by the length restriction of the Microsoft WebBrowser Control, which is max. 259).

- In the Windows NRE, an event may occur as the window.document.title attribute value is changed when loading the web page.
  The userdata delivered when handling an event should be set as conditional to separate it from the event occurring in the website itself during use.
