WebSocket NRE Support
Overview 
The WebSocket object can be usued in the NRE.
Describes whether functions (properties, methods) are supported depending on the operating system running the app.

* Please refer to the link below for whether the WebSocket object function is supported according to the browser version in the Web Runtime Environment.
https://developer.mozilla.org/en-US/docs/Web/API/WebSocket#browser_compatibility

* For detailed information on each function, please refer to the specification document below:
https://html.spec.whatwg.org/multipage/web-sockets.html#the-websocket-interface
Restrictions in Windows NRE 
- Only Windows 7 and later operating systems are supported.
- When trying to communicate with the wss protocol in the Windows 7, the server to which the private certificate is applied cannot be connected.
- The protocols(optional) parameter in Constructor is not supported.
Restrictions in iOS/iPad NRE 
The WebSocket object cannot be used when the Update Type is specified as "Update" or "Local" when packing the app.
According to WKWebView policy, files stored locally cannot handle WebSocket communication (an error occurs when executed).
Whether to support the functions below is only applicable if the Update Type is set to "Server".
