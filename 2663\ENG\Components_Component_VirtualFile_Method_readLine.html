readLine
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method is used to read the contents of a file opened in a VirtualFile up to a certain point.
Syntax 
VirtualFile.readLine([strDelimeter[, strCharset]])
Parameters
Parameters Type Description
strDelimeter String 
It specifies a delimiter that indicates where to stop reading in a file.

The default value is "\n".
strCharset String 
It specifies a character set applied to contents to read

This parameter is supported only by the Windows NRE. In mobile devices, the UTF-8 character set will be applied.
Sample Call:
var varValue ;
varValue = this.VirtualFile00.readLine();
varValue = thie.VirtualFile00.readLine("\p");
Return
Type Description
Boolean 
This method will return "false" if its parameters contain a character that is not permitted. 
Otherwise, it will return "true".
Remark 
- If this method succeeds in reading a file, it will trigger the onsuccess event
 If it fails, it will trigger the onerror event.

- This method reads data excluding the string specified by strDelimeter.

- The position of a file pointer will move once a file is read.

- You cannot use this method for a file opened in the binary mode.
   Read data is delivered as a value of the binarydata or textdata property, which both belong to the VirtualFileEventInfo object, the 'e' object of the onsuccess event.
