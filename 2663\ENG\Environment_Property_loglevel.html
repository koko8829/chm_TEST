Property Type

Description 
This property is used to set the level of information that the application outputs as log files.
Syntax 
Environment.loglevel[= strloglevel]
Setting Syntax 
loglevel ::= 'fatal' | 'error' | 'warn' | 'info' | 'debug'
 
var objEnv = nexacro.getEnvironment();
objEnv.loglevel = "warn";
"fatal" 
Only "fatal" messages are printed in the log.
"error" 
It prints the messages up to the "error" level in the descending gravity level.
"warn" 
It prints the messages up to the "warn" level in the descending gravity level.
"info" 
It prints the messages up to the "info" level in the descending gravity level.
"debug" 
It prints the messages up to the "debug" level in the descending gravity level.
Debug information includes a string output when the trace method is executed.
Remark 
- If the loglevel property value is not set, it will be set as "debug."

- If the level of the error that occurs is equal to or higher than the set value, a log will be output to a log file.
   The error levels are in "fatal" > "error" > "warn" > "info" > "debug" in the order of severity. 
   From "fatal" to "info," it is log information processed internally by the framework.

- The loglevel property operates according to the following order of priority:
   (1) the loglevel property value in nexacro_config.xml configuration file;
   (2) the loglevel property value set in the script;
   (3) the loglevel property value set in Nexacro Studio.
   However, the loglevel property value in the nexacro_config.xml configuration file will be reflected only during execution and is not returned when checking the Environment.loglevel property value in the script.

- Where log files are created:
   Windows NRE, macOS NRE: "%USERAPP%"

- Log file names
   If it is not defined in the Nexacro Config file, the log file names will be created in the form of "nexacro_"+[Protocol]+[URL]+."log."
