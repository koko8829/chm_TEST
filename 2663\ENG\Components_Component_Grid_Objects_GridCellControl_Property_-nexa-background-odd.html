Description 
The attribute to set the background area of the cell when the row index of the cell is in an uneven number.
Syntax 
GridCellControl.-nexa-background-odd
Setting Syntax 
-nexa-background-odd ::= [<bg-image> | <linear-gradient>] [<background-origin> [<background-clip>]] [<background-color>]

<bg-image> ::= 'none' | <imageurl> [<repeat-style>] [<position>]

<bg-image> 
This is a background image set with the format "'none' | <imageurl> [<repeat-style>] [<position>]" 

An image will be displayed upon the color specified by the <background-color> option.

If you specify this value as "none", no image will be displayed.
The <imageurl> option is used to specify an image.
The <repeat-style> option is used to specify the direction of displaying an image repeatedly.
The <position> option is used to specify the position of displaying an image.

You cannot set gradients and a background image simultaneously.
<imageurl> 
This value sets a background image with the format "'URL(' <Theme-Image> | <Web-Image> ')'".

It sets an image from the Web or a theme file with the format "URL ( [path of an image] )".
<Theme-Image> 
You can specify an image declared in a theme file with the format "theme://images\image name".

The relevant image must be defined in the theme file.
<Web-Image> 
You can specify an image located in the Web with the format "http://path/image name" or "https://path/file name".

You must have a Web-enabled environment.
<repeat-style> 
This is the direction of displaying an image repeatedly.

If you specify "repeat-x," an image will be repeated in the horizontal direction.
If you specify "repeat-y," an image will be repeated in the vertical direction.
If you specify "repeat," an image will be repeated in both the horizontal direction and the vertical direction.
If you specify "no-repeat," an image will not be repeated.

If you specify no value, the "repeat" option will apply.
<horizontal-position> 
This is the horizontal position of displaying an image.

If you specify "left," the left side of an image will be aligned to the left side of a GridCellControl.
If you specify "center," the center of an image will be aligned to the center of a GridCellControl.
If you specify "right," the right side of an image will be aligned to the right side of a GridCellControl.
If you specify a percentage as a value of <pos-percentage>, the image will be positioned as the value expressed in ratio.


If you do not specify <horizontal-position> while specifying <vertical-position>, the value of <horizontal-position> will be set to "center" automatically.
If you do not specify both <horizontal-position> and <vertical-position>, the value of <horizontal-position> will be set to "left" automatically.
<vertical-position> 
This is the vertical position of displaying an image.

If you specify "top," the top of an image will be aligned to the top of a GridCellControl.
If you specify "center," the center of an image will be aligned to the center of a GridCellControl.
If you specify "bottom," the bottom of an image will be aligned to the bottom of a GridCellControl.
If you specify a percentage as a value of <pos-percentage>, the image will be positioned as the value expressed in ratio.

If you do not specify <vertical-position> while specifying <horizontal-position>, the value of <vertical-position> will be set to "center" automatically.
If you do not specify both <horizontal-position> and <vertical-position>, the value of <vertical-position> will be set to "top" automatically.
<pos-percentage> 
This value indicates the horizontal/vertical position of displaying an image, expressed in ratio.

If you specify 0% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "left".
If you specify 50% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "center".
If you specify 100% in <horizontal-position>, the image will be arranged as if the value of <horizontal-position> is "right".
If you specify 0% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "top".
If you specify 50% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "center".
If you specify 100% in <vertical-position>, the image will be arranged as if the value of <vertical-position> is "bottom".

If you specify a number smaller than 0 or bigger than 100, the position will be a virtual point located outside of the GridCellControl.

If you specify this value, you cannot omit the unit "%".
<linear-gradient> 
This is background gradients set with the format "'linear-gradient(' [<angle> , ] <color-stop> ')'".

Gradients will be displayed upon the background color specified by the <background-color> option.

You can specify the direction and colors of gradients with the format "linear-gradient ( [direction, color1, color2, ...] )".

You cannot set gradients and a background image simultaneously.
<angle> 
It specifies the direction of gradients.

If you specify "to left," gradients will run from right to left.
If you specify "to right," gradients will run from left to right.
If you specify "to top," gradients will run from bottom to top.
If you specify "to bottom," gradients will run from top to bottom.
If you specify "to left top," gradients will run from bottom right to top left.
If you specify "to left bottom," gradients will run from top right to bottom left.
If you specify "to right top," gradients will run from bottom left to top right.
If you specify "to right bottom," gradients will run from top left to bottom right.

If you specify no value, the "to bottom" option will apply.
<color-stop> 
It specifies colors for the beginning, middle and ending points of gradients with the format "<startcolor-stop> [<listcolor-stop>] <endcolor-stop>"

A beginning and ending points will be determined according to the value of <angle>.
Assuming that <angle> is specified as "to left," for example, the right side of a GridCellControl will be a beginning point while the left side will be an ending point.
Assuming that <angle> is specified as "to right," for example, the left side of a GridCellControl will be a beginning point while the right side will be an ending point.

You can specify more than one color for the middle point of gradients. The specified colors will be equally distributed if you omit the value of <percentage>.
<startcolor-stop> 
It specifies the color of the beginning point of gradients with the format "<color> [<percentage>'%']".

A beginning point will be determined according to the value of <angle>.
If you specify the <percentage> of a beginning point as 0 or larger, a same color will be displayed from the beginning point to the specified point.
<endcolor-stop> 
It specifies the color of the ending point of gradients with the format "<color> [<percentage>'%']".

An ending point will be determined according to the value of <angle>.
If you specify the <percentage> of an ending point as 100 or smaller, a same color will be displayed from the specified point to the ending point.
<listcolor-stop> 
It specifies the colors of the middle points of gradients with the format "<color> [<percentage>'%"] [,<listcolor-stop>]*".

You can specify multiple colors to apply between a beginning and ending points by dividing them with commas.
A following declared color must have a bigger <percentage> value than the preceding declared color.
<color> 
It specifies the colors of gradients with color names or color codes.
<percentage> 
This is the positions where the colors specified by <color> will apply. Those positions are specified in percentage.

"0%" indicates the beginning point determined according to the value of <angle>.
"100%" indicates the ending point determined according to the value of <angle>.

A percentage smaller than 0 or bigger than 100 indicates a virtual point located outside the GridCellControl.
Assuming that the value is 150%, for example, the value of <color> will apply to the point, which is calculated as the middle point and located within the GridCellControl.

If you omit the value of <percentage>, the value of <color> will apply to a proportionate point.
You cannot omit the unit % when you enter a value.
<background-origin> 
This is the area that represents the entire background.

If you specify "border-box", the entire area includes the border.

If you specify "padding-box", the entire area includes the inside of the border.
In this case, the relevant area serves as a base for the beginning and end of a gradient as well as the repetition of a certain background pattern.

The default value is "padding-box".
<background-clip> 
This is the area where the background is actually displayed.

If you specify "border-box", the area includes the border.
If you specify "padding-box", the area includes the inside of the border.

If you omit this value, the value of <background-origin> will be applied instead.
If you even omit <background-origin>, "border-box" will be applied.
<background-color> 
This value is a color to display with a color name or color code.

If the value of <bg-image> or <linear-gradient> is specified, the background color will be displayed under the background image or gradients
A background color applies to the transparent part of an image specified by <bg-image>. It also applies to the empty part in a GridCellControl, which is produced as a result of applying an image specified by <bg-image>. In this case, the image is smaller than the GridCellControl.

You cannot set gradients and a background image simultaneously.
<NamedColor> 
You can specify a predefined color name in the form of a string like "red" and "blue".
<NumericColor> 
This is a code that indicates the color to display.

The code is a hexadecimal values that begins with "#" and comprises the following 6-digit value (eg. #FF00FF).
You can specify a color code with the rgb() method whose parameters are red, green and blue values. (eg. rgb(255,0,255))
You can specify a color code with the rgba() method whose parameters are red, green, blue and alpha values. (eg. rgba(255,0,255,0.3))
You can specify a color code with the hsl() method whose parameters are hue, saturation and lightness values. (eg. hsl(120,50%,50%))
You can specify a color code with the hsla() method whose parameters are hue, saturation, lightness and alpha values. (eg. hsla(120,50%,50%,0.8))
Remark 
- It is an attribute that can be set in the XCSS only and the value cannot be changed by a script.
   - It is set as undefined and operated as "transparent" if the attribute value is not set in the XCSS.

- The user prefix defined in the Services of the TypeDefinition area cannot be used as the image path in the theme or XCSS.

- The criterion area and the display area of the background is set as <background-origin> and <background-clip>, respectively.
   Cell does not display all borders (top/right/bottom/left) but operates to connect the border with nearby cells with the right and bottom only.
   Accordingly, the left of a specific cell is replaced with the right of the left cell, and the top is replaced with the bottom of the top cell.
   These properties should be considered when setting the background area.

- The <position> value should be set as "center" or "center center" to locate the background image to the center.

- The image can be displayed on the background color if both the background image and background color are set.
   The gradation is displayed on the background color if both the gradation and background color are set.

- The gradation and the background image cannot be set at the same time.
   A malfunction may occur when setting both values at the same time.

- This attribute is applied only to the cell belonging to the body area of the Grid.
