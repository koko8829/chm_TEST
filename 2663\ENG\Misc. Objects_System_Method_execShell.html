execShell
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method is used to execute a program in the Windows shell.
Syntax 
system.execShell( strShellCmd )
Parameters
Parameters Type Description
strShellCmd String 
Sets the value to pass to the window shell.
The available formats are shown below.

1. File path + File name.
Specify the file path and file name of the user's PC and run in the connected app.
You can set it to Alias or specify a direct file path.

2. URL
Strings starting with "http://" or "https://" are passed as URL values and run in the default web browser.

3. Email
Strings starting with "mailto:" will run in the default email app.
Sample Call:
// file name
system.execShell("%MYDOCUMENT%test.png"); // alias
system.execShell("C:\\Users\\test\\Documents\\test.png");
system.execShell("file://C:\\Users\\test\\Documents\\test.png");

// url
system.execShell("http://www.test.com");
system.execShell("http://www.test.com/test.png");

// email
system.execShell("mailto:user00@test.com?subject=TestSubject&cc=user01@test.com&body=TestBody");
Return
None
Remark 
- The execShell method only delivers the set value to the window shell and does not run a specific program directly.
  Therefore, it may operate differently depending on the user's PC environment settings.

- For security reasons, the set value is not passed to the window shell in the following cases:
  
1. Specific executable file extensions (38 types)
Restrict extensions that can install or run malicious code on a user's PC.
this method will not be executed if a file has one of the extensions listed below:
bat, bin, cmd, com, cpl, dll, exe, gadget, hta, inf1,
ins, inx, isu, job, js, jse, lnk, msc, msi, msp,
mst, paf, pif, ps1, reg, rgs, scr, sct, shb, shs,
u3p, vb, vbe, vbs, vbscript, ws, wsf, wsh

2. If there is no file on the user's PC
If a file name is set as a parameter value, it checks whether the file exists in the user's PC, and if not, do not forward the set value to the window shell.

3. If there is no extension
It cannot be executed by passing the file name without extension or the system registration program name as a parameter value as shown below.
[X] system.execShell("notepad")
