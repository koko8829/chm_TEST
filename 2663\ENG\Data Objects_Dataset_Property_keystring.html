Description 
This property specifies an expression that is used to group or sort the data loaded onto a Dataset.
Syntax 
Dataset.keystring[= strKeystring]
Setting Syntax 
strKeystring ::= <Key> [',' <Key>] [',' <OrderKey>]

<Key> ::= [<group-type> ':'] <group-col>

<group-type> ::= 'G' | 'S'
<group-col> ::= [<sort-type>] <column-id> [[','] <group-col>]

<sort-type> ::= '+' | '-'

<OrderKey> ::= 'O:'<column-id>'['<locale code>']'[<locale option>]
 
this.Dataset00.keystring = "column0";
this.Dataset00.keystring = "G:column0";
this.Dataset00.keystring = "G:+column0-column1";
this.Dataset00.keystring = "S:column0+column0";
this.Dataset00.keystring = "G:+column0, S:-column1";
this.Dataset00.keystring = "S:column0, O:column0['en-US']";
this.Dataset00.keystring = "S:column0-column1, O:column0['en-US']{ 'sensitivity' : 'base', 'ignorePunctuation' : true }";
<group-type> 
This an arrangement method.

Specify "G" for grouping.
Specify "S" for sorting.

The default value is "G".
<sort-type> 
This is how to sort data.

Specify "+" for sorting data in ascending order.
Specify "-" for sorting data in descending order.

The default value is "+".
<column-id> 
This is the ID of a base column for grouping data.
<OrderKey> 
Determines the sorting method of the base column according to the set locale code and locale option.
Only column id set in <group-col> can be set as column id in <OrderKey>.
<locale code> 
Enter localale information directly and determine the sorting method based on the entered localale information.
It can be entered according to the BCP 47 Language Tags standard.
<locale option> 
Detailed options can be set to JSON format depending on the setting language.
- sensitivity: Sets options for string sorting. The default value is "variant".
  * base: Treat the default character as different strings.
  * accent: The default character is the same, but if there is an accent mark, treat it as something else.
  * Case: If the default character or case is different, treat it as something else.
  * Variant: Treat the default characters as different if they have the same accent mark and if the case is different.
- ignorePunctuation: Sets whether to ignore punctuation. The default value is false.
- numeric: Sets whether to treat string type numbers as numbers. The default value is false.
- caseFirst: Sets whether uppercase or lowercase letters should be sorted first. The default value is false.
   * upper: Uppercase letters have higher priority.
   * lower: Lowercase letters have higher priority.
   * false: Process according to locale preference.
Remark 
- If this property is set, the onrowsetchanged event will occur after grouping/sorting is done according to the specified expression.
In this case, the reason property of the onrowsetchanged event's EventInfo object has "REASON_SORTGROUP" as its value.

- The below descriptions are tips for using the keystring property.

  • You can set levels for groups by using commas.
For example,  "G:column1,column2" indicates that you will form the "column1" group first and then the "column2" group within column1.
Since a column name can have a blank, there must be no blank in front of or behind a comma,
    • 

  • If a comma is not used, you can merge more than one columns into one group key.
For example, "G:+column1-column2" indicates that you will form one group by combining column1 and column2.
    • 

  • If "S:" is used as a first letter, you can sort data without grouping it.
For example,  "S:+column1-column2," indicates that you will sort column1 and column2 in ascending and descending order respectively.
If you omit <group-type>, the G option will be applied.

- After grouping, logical records are created in a Dataset.
With logical records, you can determine what type of values a new column has by using the prop property.
The prop property has the following values: Count, Sum, Max, Min, Avg, and Text.
A column specified by the keystring value cannot be designated by the prop value.

- The keystring property will be influenced by the loadkeymode property if the application loads a Dataset or conducts a transaction.
 Refer to the document of the loadkeymode property for further explanation.

- With the datatyperule property set to "2.0", you should be careful when sorting data because the new data validation rule is applied.

  • If the column type is "STRING", the following order will be applied: EmptyString("") < [Value] < undefined==null==NaN
    • 

  • If the column type is "INT", "FLOAT" or "BIGDECIMAL", the following order will be applied: -Infinity < [-Number] < 0 < [+Number] < Infinity < EmptyString("") < undefined==null==NaN
    • 

  • If the column type is DATE", "TIME" or "DATETIME", the following order will be applied: EmptyString("") < [Date] < [Invalid Date] < undefined==null==NaN

- When setting OrderKey, there are restrictions as follows:

  • If the browser does not support the value set by locale code, it operates based on the operating system or browser language information and displays a warning message.
    • 

  • If any of the attributes set in locale code overlap with those set in locale option, the locale option setting value is applied.
    • 

  • Even if the collation attribute is set, the locale code setting value is applied when processing the corresponding column.
For example, if the collation attribute value is set to "en-US" and the keystring attribute value is set to "S:column0,O:column0['ja-JP']", the "ja-JP" set by keystring is applied as the local code.
