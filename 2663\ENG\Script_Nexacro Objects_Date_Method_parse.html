Description 
The method to return the time between midnight of January 1, 1970 to the relevant date in milliseconds by analyzing the string including the date.
Syntax 
Date.parse(strDate)
Parameters
Parameters Type Description
strDate String 
It is a string representing a date.

Strings in invalid date format are treated as errors.
Sample Call:
var objDate = new nexacro.Date();
var nMs = objDate.parse("2021-06-29T11:00:00.000Z");
Return
Type Description
Number 
It returns the time in milliseconds between midnight on January 1, 1970 and the specified date.

It returns NaN if it is treated as an error in an invalid date format.
Remark 
- This method accepts a date string that complies with ISO 8601 or RFC 2822 of IETF. 

 * ECMA standard
  [http://www.ecma-international.org/ecma-262/5.1/#sec-15.9.1.15]
* Format according to ISO 8601: YYYY-MM-DD'T'hh:mm:ss.sss'Z'
  [http://www.iso.org/iso/home/standards/iso8601.htm]
* Format according to IETF of RFC2822: 'May 21, 2012', 'Mon, 21 May 2012 03:20:00 +0900'
  [https://datatracker.ietf.org/doc/html/rfc2822#page-14]
* Formats supported by Mozilla web browsers
  [https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date/parse]

â—† Restrictions in WRE

- ECMAScript 5 standard specifies to follow the ISO 8601 date format.
  However, the previous ECMA standard specified to interpret the execution result of the toString method of the Date standard built-in object, so the behavior may differ depending on the browser and an error may occur.

- To get the same result regardless of browser, the date format string must be used as shown below:
  YYYY-MM-DD (ISO 8601 date format, e.g. 2021-06-29)
  YYYY-MM-DD'T'hh:mm:ss.sss'Z' (ISO 8601 combined UTC date and time, e.g. 2021-06-29T11:00:00.000Z)
  MMM DD, YYYY (e.g. May 21, 2012)
  EEE, DD MMM YYYY hh:mm:ss Z (e.g. Mon, 21 May 2012 03:20:00 +0900)
  EEE, DD MMM YYYY hh:mm:ss z (e.g. Wed, 09 Aug 1995 00:00:00 GMT)
  EEE, DD MMM YYYY hh:mm:ss (e.g. Wed, 09 Aug 1995 00:00:00)
