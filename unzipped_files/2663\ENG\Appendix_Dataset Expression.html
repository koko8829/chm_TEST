Dataset Expression
Overview 
A Dataset expression script is used to program an action designed only for a specific row of a Grid-bound Dataset.

A Dataset expression script is usually defined by some properties of a Grid component or Grid cell when those properties can set an expression, which is defined with the prefix “expr:”.
The expression script can be used as an argument for some methods of a Dataset including getCaseSum(), filter() and findRowExpr().

Using an expression, you can define an action designed only for a specific row of a Grid-bound Dataset. Then, you can display or use the result of conducting the expression.
A Dataset expression script is written according to the syntax rule of the Nexacro Form script.
Additional Expression & Objects 
When writing a Dataset expression, you are provided with the below properties and reserved words to deal with a Dataset component and Dataset column.

1. Additional Global Properties

 1.1. Object property

  1.1.1 dataset property

       This property is used to designate the Dataset bound to a component that holds an expression.
	   
       ex)
       <Cell text=”expr:dataset.rowcount”/>       // “dataset” indicates the bound Dataset
       <Cell text=”expr:dataset.parent.func1()”/> // this executes the func1 function defined in the parent Form that includes the Dataset

  1.1.2 comp property

       This property is used to designate the component itself that holds an expression.

       ex)
       <Cell text=”expr:comp.parent.func1()”/>    // this executes the func1 function defined in the parent Form that includes the component

 1.2. Dataset Row Iterator Properties 

  1.2.1 currow property

       This property indicates the index of the Dataset row that executes the current expression.
       
       ex) 
       <Cell text=”expr:dataset.getColumn(--currow, 'Column0') - Column0”/>  // this calculates the difference between the previous and current rows

  1.2.2 this property

       This property is used to designate the object that holds an expression.

       ex)
	   <Band id="head" tooltiptext=”expr:this.col”> // “this” indicate the band object
       <Cell text=”expr:this.col”/>                 // “this” indicate the cell object

2. Column Identifier Reference

    You can use a Dataset column ID directly in an expression to obtain the column value of the row that executes the expression.
    However, the ID will be converted to a primitive value to conduct the basic arithmetic operations or comparison operation.
Execution Context 
As a Dataset expression is executed based on a Form context, it can use all the properties and functions that are available in a Form.

While a Dataset expression works like an eval script, a variable declared in a Dataset expression does not affect a Form context because the Form script and expression script are contextually separated.
However, a variable declared in a Form context can be read or used in a Dataset expression script. Moreover, a Form context will be affected if the value of such a variable changes in a Dataset expression script.
