fireEvent
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This method is used to fire an event by calling the handler defined in the event.
Syntax 
EventObject.fireEvent( objFireComp, objEventInfo [, bCancelable] )
Parameters
Parameters Type Description
objFireComp Object 
This is an object that will be delivered to the event handler.

Normally, this object constitutes an object that fires the relevant event.
objEvent Object 
This is an EventInfo object that will be delivered to the event handler.
bCancelable Boolean 
This parameter determines whether to cancel the execution of handlers depending on a return value.

With this parameter set to "true", all the following handlers will be canceled if a handler returns "false".
With this parameter set to "true", the following handler will be executed if any of a handler returns "true".

With this parameter set to "false", the following handler will be executed regardless of a return value.

The default value of this parameter is "false".
Sample Call:
var objEventInfo = new ClickEventInfo();

var bRtn = this.Button00.onclick.fireEvent( this.Button00, objEventInfo, true );
Return
Type Description
Boolean 
This method will return "false" if a handler returns "false" with the bCancelable parameter set to "true".
This method will return "true" if no handler returns "false" with bCancelable set to "true".

This method will return the value that is returned by the last handler if bCancelable is set to "false".
This method will return an empty value if bCancelable is set to "false" and the last handler does not return any value.
Remark 
- This method is used to fire an event in a user component like a composite component.

- If there are two or more handlers are defined in an event, the handlers will be executed in the order that they are defined.

- Normally, this method is intended to receive information that a handler has canceled the execution of the relevant event.
 That is, this method is intended for the case where a handler returns "false" with the bCancelable parameter set to "true".
Example
Script 
this.fn_onclick_01 = function(obj:nexacro.Button,e:nexacro.ClickEventInfo)
{
    trace("Event 01");
    return false;
};

this.fn_onclick_02 = function(obj:nexacro.Button,e:nexacro.ClickEventInfo)
{
    trace("Event 02");
    return false;
};

trace(this.Button02.addEventHandler("onclick", this.fn_onclick_01, this));
trace(this.Button02.addEventHandler("onclick", this.fn_onclick_02, this));

trace(this.Button02.onclick.fireEvent(this.Button02, e, false));
trace(this.Button02.onclick.fireEvent(this.Button02, e, true));
