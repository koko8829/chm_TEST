Description 
The method to import the file as DataSet.
Syntax 
ExcelImportObject.importData( strSource,strRange,strOutDatasets [,strArgument] )
Parameters
Parameters Type Description
strSource String 
You can specify the location of a file that you want to import in the form of a URL.

If the importfilemode property is set to "server", you can specify a location with the format "http://" or "https://".
If you use a relative path with importfilemode set to "server", the path will be based on the value of the importurl property.
If importfilemode is set to "local" or you do not set this parameter, a FileDialog will appear to allow you to select a file.
strRange String 
Set the area to be imported from the set file in the strSource to the DataSet as “Sheetname!cellarea:cellarea”, or as “startarea:endarea” form if the format of the file to be imported is CSV or TXT.

Follow the format below and set repeatedly if you need to specify multiple areas.

strRange ::= '[Command=<strCommand>; Output=<dsName>; Head=<strArea>; Body=<strArea>]'

<strCommand> ::= 'getsheetlist' | 'getsheetdata'
 The "getsheetlist" setting imports the sheet list from the file set in the strSource.
  The "getsheetdata" setting imports the <strArea> data from the file set in the strSource.
  It is applied as the "getsheetdata" when the value is omitted.

<dsName> 
  It is the variable name which has the imported data.
  It is used as the right assignment variable in the strOutDatasets.
  It is automatically set as "output1", "output2" ... if the value is omitted

<strArea> ::= 'Sheetname!cellarea:cellarea'
  Set the data area to be imported.
  The area defined in the “Head” item is used as the Column name of the DataSet when importing.
  It is automatically set as "Column0", "Column1" ... if the “Head” item is omitted.
  The area defined in the “Body” item is imported as the row data of the DataSet.
  All records are imported when the “Body” item is omitted.
  * The first sheet becomes the target if the sheet name is omitted.
  * Both "Head" and "Body" should be set if sheets other than the first sheet are designated.
    If set as below, the Head is processed in the first sheet and only the Body is processed in the designated sheet.
    An error may occur if there is no cell area set from the Body in the Head.
    [Command=getData;Output=output2;Body=Sheet2!A1;]

* It is applied as below if the format of the file to be imported is CSV or TXT.

<strCommand> is not supported.
<strArea> ::= StartCol[,StartRow[:EndCol[,EndRow]]]
  It is processed as below if the StartRow, EndCol, and EndRow are omitted in the “Head” item.
  - StartRow=0, EndCol=ColumnCount-1, EndRow=0
  In the “Head” item, the EndRow setting value is ignored and it is operated based on the StartRow setting value.
  It is processed as below if the StartRow, EndCol, and EndRow are omitted in the “Body” item.
  - StartRow=0, EndCol=ColumnCount-1, EndRow=RowCount-1
strOutDatasets String 
This parameter specifies the ID of a Dataset that stores the imported data with the format "Dataset name=variable name".

You can set one or more Datasets by using whitespaces as separators.
The variable name on the right-hand side of the equation adopts the variable name defined by the strRange parameter or an automatically-created name such as "output1".
strArgument String 
You can specify information needed for import.

You can set one or more pairs of arguments by using whitespaces as separators.
If you import an Excel file that is locked with a password, you must enter an argument with the format ""filepassword=the password of the Excel file" to succeed in the import.
Sample Call:
this.ExcelImportObject00.importData("http://localhost:8080/xeni/XImport/my.xlsx","Sheet00!D15:E28","Dataset03=output1");
this.ExcelImportObject00.importData("","Sheet00!D15:E28","Dataset03=output1","filepassword=strpassword");
this.ExcelImportObject00.importData("", "[Command=getsheetlist;Output=sheetlist][Command=getsheetdata;Output=output1;Head=!A1:K1;Body=!A2:K15][Output=output2; Body=Sheet1!B7:Z207]", "ds_list=sheetlist, ds_Data1=output1, ds_Data2=output2");
Return
None
Remark 
- Entering ESC while importing stops the import.

- It sends the file located in the Local to the server set in the importurl attribute and receives by dataset like a transaction.

- The difference with the importDataEX() method is whether there is an ID parameter to separate the import.

- Objects (images, charts, formulas, etc.) included in the file to be imported are not included in the import target, and if they are included, the object is ignored and processed.
