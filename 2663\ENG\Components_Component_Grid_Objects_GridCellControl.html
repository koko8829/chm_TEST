Description 
A cell is a minimum unit through which a Grid prints data. A cell is an object that must take its unique position within a band.
Remark 
- Each cell has a unique index value counted within the relevant band. 
 An index number is counted from the left to the right side of the relevant band.
When there is a sub-row, index numbers will be allocated to the cells in the parent row first, also from the left to the right side. Then, the cells in the sub-row will be given the following index numbers.
As for a merge cell or sub-cell, the cell will take the index number of the first cell among the separate cells forming the merged cell.

- Cell index is not correlated with the number of data items contained by a Dataset. Rather, it is related to the format defined by the Grid.

- You can modify sub cells individually by using the method getSubCellProperty().
 Each sub cell has their own sub cell index.
 A merge cell is treated as one cell, and you cannot modify its child cells individually.

- Some properties of a cell can be bound to a Dataset column or set with an expression.
 For such an expression, you can use the bound Dataset, column ID as well as the functions defined in accessible areas.

- The below list presents the reserved words that represent each designated value.
  > this: the Grid cell where the expression is defined. (It does not indicate the Form.)
  > comp: the Grid that contains the cell
  > dataset: the Dataset bound to the Grid
  > currow: the index number of the row that the expression calculates
  > [Column ID]: the value of the Dataset column bound to the row that the expression calculates
  > The properties and methods of the Dataset bound to the Grid
      ex) "dataset.rowcount", "dataset.colcount", "dataset.rowposition", "dataset.getCount()"
