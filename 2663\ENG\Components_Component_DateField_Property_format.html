Description 
This property is used to set the format of the date displayed in the edit window when the DateField component is focused.
Syntax 
DateField.format [=strFormat]
Setting Syntax 
strFormat ::= <maskchar> | <dateStyle, timeStyle>

<maskchar> ::= [<Year>] [<Month>] [<Day>] [<Week>] [<Hour>] [<Minute>] [<Second>] [<MiliSec>] [<AM/PM>]


* <maskchar> is a combination of mask characters for entering a date and time, where each element can be reordered and omitted.
 
this.DateField00.format = "yyyy-MM-dd";
this.DateField00.format = "yyyy-MM-dd HH:mm:ss";
this.DateField00.format = "SHORTDATE, LONGTIME";
<Year> 
Set the mask characters representing the year.

When set to "yyyy," all four digits of the year will be displayed.
(When "yy" is set, it will operate the same way as "yyyy.")
<Month> 
Set the mask characters representing the month.

When it is set as "M," January to September will be displayed as one digit and October to December will be displayed as two digits.
When it is set as "MM," all months will be displayed as two digits, and January to September will be displayed with "0" as the first digit.
(When it is set as "MMMM" or "MMM," it will operate the same way as "MM.")
<Day> 
Sets the mask characters representing the day.

When it is set as "d," the 1st to 9th will be displayed as one digit, and the other days will be displayed as two digits.
When it is set as "dd," all days will be displayed as two digits, and the 1st to 9th will be displayed with "0" as the first digit.
<Week> 
Set the mask characters representing the day of the week.

When it is set as "ddd,," the abbreviated name of the day  of the week will be displayed.
When it is set as "dddd," the full name of the day of the week will be displayed.
<Hour> 
Sets the mask characters representing the hour.

When it is set as "h," 1:00 to 9:00 will be displayed as one digit based on the 12-hour clock, and the other hours will be displayed as two digits.
When it is set as "hh," all hours will be displayed as two digits based on the 12-hour clock, and 1:00 to 9:00 will be displayed with "0" as the first digit. 
When it set as "H," the hours will be displayed based on the 24-hour clock, with 1:00 to 9:00 displayed as one digit, and all the other hours displayed as two digits.
When it is set as "HH," ,the hours will be displayed based on the 24-hour clock, with 1:00 to 9:00 will be displayed with "0" as the first digit. 

* If you manually enter the time after setting "h" or "hh," it will not be possible to distinguish between AM and PM. 
   By setting the <AM/PM> values at the same time, it will be possible to designate AM or PM.
<Minute> 
Set the mask characters representing  the minutes.

When it is set as "m," minutes from 1 to 9 will be displayed as one digit, and the rest will be displayed as two digits.
When it is set as "mm," it will all be displayed as two digits, and minutes from 1 to 9 will be displayed with "0" as the first digit.
<Second> 
Set the mask characters representing the seconds.

When it is set as "s," seconds from 1 to 9 will be displayed as one digit and the rest will be displayed as two digits.
When it is set as "ss," it will all be displayed as two digits, and seconds from 1 to 9 will be displayed with "0" as the first digit.
<MiliSec> 
Set the mask characters representing milliseconds.

When it is set as "sss," all milliseconds will be displayed as three digits, and 1 to 99 ms will be displayed with "0" as the first digit.
<AM/PM> 
Set the mask characters indicating AM or PM.

When it is set as "aa," AM and PM will be displayed according to the locale.
If the <Hour> value is set as "H" or "HH" and the time value is entered, the <AM/PM> value will be set automatically and cannot be changed.
<dateStyle, timeStyle> 
Set as the style options used by the ECMA Intl API.

"SHORTDATE, SHORTTIME" - dateStyle: 'short', timeStyle: 'short'
"SHORTDATE, MEDIUMTIME" - dateStyle: 'short', timeStyle: 'medium'
"MEDIUMDATE, SHORTTIME" - dateStyle: 'medium', timeStyle: 'short'
"MEDIUMDATE, MEDIUMTIME" - dateStyle: 'medium', timeStyle: 'medium'
"LONGDATE, SHORTTIME" - dateStyle: 'long', timeStyle: 'short'
"LONGDATE, MEDIUMTIME" - dateStyle: 'long', timeStyle: 'medium'
"FULLDATE, SHORTTIME" - dateStyle: 'full', timeStyle: 'short'
"FULLDATE, MEDIUMTIME" - dateStyle: 'full', timeStyle: 'medium'
Remark 
- If the format property value is not set, the format property value will be set according to the inputtype property value.

- The data format displayed on the screen when the DateField component is not focused is set in the displayformat property.

- If you set a character other than the defined mask character, it will be displayed as is on the screen and the character cannot be edited.


◆ Mask Characters That Cannot Be Edited

- <Week> mask characters can be displayed in the edit window, but cannot be modified.
   It will be automatically reflected according to the selected date value.

- When setting the <Hour> mask characters based on 24 hours, <AM/PM> mask characters can be displayed in the edit window, but cannot be modified.
   It will be automatically reflected according to the selected time value.
