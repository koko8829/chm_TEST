Description 
This method is used to export all the items set by an ExcelExportObject.
Syntax 
ExcelExportObject.exportData( [strParam, strParamDS, nOrgValue, bFileDown] ) ;
Parameters
Parameters Type Description
strParam String 
This is a string that will be delivered to the server as a parameter.

This parameter is an extensive function used to deliver the same value as the DB key.
strParamDS String 
This parameter is used to set a Dataset delivered to the server, with the format "Param_DataSet=export_DataSet".

This parameter is an extensive function used to send a Dataset to equip the server with additional features, which vary according to development environments.
nOrgValue Number 
This parameter is used to determine the targets to export as cell values.

If you specify 0 or false, the text items displayed in cells will be exported as values.
If you specify 1 or true, the column value of the bound Dataset will be exported as values.
If you specify 2, this parameter will work as if it is set to 0 when the displaytype property is set to "date". When displaytype is set to a value other than "date", this parameter will work as if it is set to 1.

The default value of this parameter is 0.

* For the cells set by the valuetypecellindex property, the column values bound to those cells will be exported regardless of the value of this parameter.
 For those are not,cell values will be exported according to the value of this parameter.

* You are recommended to use the valuetypecellindex property and exportDataEx() method to maintain the consistency in export results.
bFileDown Boolean 
This parameter determines whether to download the file created as the export.

For "true", the exported file will be downloaded automatically.
For "false", the exported file will not be downloaded automatically.

The default value of this parameter is "true".
Sample Call:
var nCount = this.ExcelExportObject00.exportData();
Return
Type Description
Number 
This method returns the number of items for which an export request has been successful.

If there is no item to export, 0 will be returned.

If arguments are invalid, -1 will be returned.
Remark 
- If the exportmessagealert property is set, export will begin after the alert box is closed.

- Events occurs during export as set by the exporteventtype property.

- If you set the bFileDown parameter to "true", a pop-up for asking about download may appear depending on environments.

- The onerror event will be fired if an error occurs during export.

- Although a cell's displaytype property is set to "decoratetext", decorations by tags will not be applied when the cell is exported because tags are HTML.
   > If the cell is set by the valuetypecellindex property, the tags in the cell will be exported as text.
   > If the cell is not set by valuetypecellindex, the nOrgValue parameter will determine whether the tags are exported as text or are removed before the export.

- Only CSS-related properties set for a Grid cell are applied when the cell is exported.

- The chunked data transfer with the transferrowcount property set is performed in the asynchronous manner.
 Be careful that changes in the Grid's properties or Dataset's data during the chunked transfer can apply to the exported data.
 If the formatid property is changed after the exportData() method, for example, the format of the pending data chunk can be changed.
