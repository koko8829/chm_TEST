Description 
Itâ€™s an object that saves the data in a table form.

It is closely related to the engine, and the Dataset form is used for data used internally.
It shares data changes using the Dataset bound with the component. 
It is used in a form sending and receiving data even when communicating. 

It supports the method to add/delete/edit the column or records or to group, arrange, filter, or apply a statistical method (Sum, Max, Min, Avg, Count),
or it supports the function to merge, add, or copy other dataset.

The record of the Dataset can be filtered by selecting Expression from the filterstr attribute.
It can bind the data in a group or arrange using the keystring attribute.
Remark 
- The Dataset supports methods such as getDeletedRowset(), getOrgColumn(), etc. that can check the content of the edited/deleted record before a change.
   Also, a separate method is provided that can control the filtered records and unfiltered records.

- Events occur in the Dataset depending on the situation.
  In particular, since the onload event occurs in various situations, it should be separated and processed by the reason attribute of the event object.

- The Column name of the Dataset cannot include spaces.

- The Dataset can be assigned as a parameter in the method such as the copyData(), assign(), etc., of the Dataset.
   At this time, if there is no Dataset assigned in the Form, it is searched for from the upper Form.
   The SubForm.Dataset object must be explicitly given as a factor to assign the Dataset of the subform.

Name Description
