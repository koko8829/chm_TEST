Description 
This method returns the value of a certain property that applies to a certain cell.
Syntax 
Grid.getCellProperty( enumBand, nCellIdx, strPropID )
Parameters
Parameters Type Description
enumBand String 
This parameter is used to specify the band that includes the target cell.

If you specify "head", the head band includes the cell.
If you specify "body", the body band includes the cell.
If you specify "summary", the summary band includes the cell.
nCellIdx Number 
This is the index number of the target cell.
strPropID String 
This is a string that represents the name of the target property.
Sample Call:
var varProperty = this.Grid00.getCellProperty( "head", 0, "color" );
Return
Type Description
String 
This method returns the value of a certain property that applies to a certain cell.
Remark 
- This method can deal with only the properties set through a script or Nexacro Studio. 
 The property values defined in a theme file can be applied, but cannot be saved to the properties.

- Each of the head, body and summary bands of a Grid has its own row format, and a row can have multiple sub-rows.
The number of the items in the bound Dataset does not matter.
 The index number of a cell is counted as follows if the cell belongs to a band that holds sub-rows: sub-row index number * the number of columns + the column value of the cell.
  When cells are merged, the index number of a cell will decrease as much as the number of merged cells.

- In each band, cell index numbers are counted in the direction from top-left to bottom-right.
 The index number of an entire merged cell equals the index number of the first among the separate cells forming the merged cell.

- While sub-cells basically belongs to one merged cell, each of them retains their own properties.You can obtain a property's value of such a sub-cell by calling the getSubCellProperty() method.

- If you have not changed the value of the col property of a GridCellControl, the cell index will begin from 0.
