Description 
The TextArea is a component used to receive multiple-line string input from the user.
Remark 
- It can be used to output the string by the user.

- The LF("\n") characters in the TextArea are treated as line breaks.
   The Enter key is treated as a line break when entering text in the TextArea.

- The Undo/Redo functions can be used on the value being entered in the TextArea.

- The Edit should be used to receive input of a single line of string.

- The MaskEdit should be used to receive input of characters or numbers in a specific form.

- The paste function does not work if entering Japanese with Japanese IME (combining and not confirmed yet).
Structure
Contents Sizing 
The below table describes how to size a TextArea.
NULL 
TextArea Component  * An hscrollbar control will appear if the width of the TextArea is smaller than the width of the longest text line. * A vscrollbar control will appear if the height of the TextArea is smaller than the total height of all text lines.
Width 
The value of the width property of the TextArea is applied. If the length of the text in the component is longer than the width, the text will be hidden from sight partially.
Height 
The value of the height property of the TextArea is applied. If the height of the text in the component is taller than the height, the text will be hidden from sight partially.
Basic Key Action
Action Key Description
Move focus Tab 
The focus moves to the next component according to the tab movement order.
Move caret Arrow keys 
A caret will move to the direction of an entered arrow key when the caret is displayed.
Select text Shift + arrow keys 
A caret will select text to the direction of an entered arrow key when the caret is displayed.
Scroll TextArea Ctrl + arrow keys 
Entering this key combination will scroll a TextArea to the direction of an entered arrow key when the TextArea is scrollable.
Accessibility Key Action 
This is a key action to be applied when the accessibility function is activated.
Action Key Description
Move focus Tab 
The focus moves to the next component according to the tab movement order.
Move selection box Up/Down arrow key 
If a caret is placed on the first line, pressing the up arrow key will move the selection box to the previous component according to the tab order. If a caret is placed on the last line, pressing the down arrow key will move the selection box to the next component according to the tab order.
Move caret Up/Down arrow key 
A caret will move to the direction of an entered arrow key when the caret is displayed. If a caret is placed on the first or last line, pressing the up/down arrow key will move the selection box.
Move caret Left/Right arrow key 
A caret will move to the direction of an entered arrow key when the caret is displayed.
Select text Shift + arrow keys 
A caret will select text to the direction of an entered arrow key when the caret is displayed.
Scroll TextArea Ctrl + arrow keys 
Entering this key combination will scroll a TextArea to the direction of an entered arrow key when the TextArea is scrollable.
Constructor 
TextArea.init( strName, vLeft, vTop , vWidth, vHeight [, vRight, vBottom, [vMinWidth, [vMaxWidth, [vMinHeight, [vMaxHeight]]]]] )
Parameters Type Description
strName String 
It sets the ID of a TextArea in the form of a string.
vLeft String, Number 
It specifies the value of the left property of a TextArea in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the right property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vTop String, Number 
It specifies the value of the top property of a TextArea in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the bottom property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vWidth String, Number 
It specifies the value of the width property of a TextArea in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vHeight String, Number 
It specifies the value of the height property of a TextArea in pixel or percent (%).

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be specified in pixel while the base component will be ignored.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vRight String, Number 
It specifies the value of the right property of a TextArea in pixel or percent (%).

This parameter will be ignored if both the vLeft and vWidth parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the left property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the width property of the base component.
vBottom String, Number 
It specifies the value of the bottom property of a TextArea in pixel or percent (%).

This parameter will be ignored if both the vTop and vHeight parameters are all specified.

* When you include a base component in the argument:
If you use a pixel as an unit, this parameter will be determined based on the top property of the base component.
If you use a percentage (%) as an unit, this parameter will be determined based on the height property of the base component.
vMinWidth String, Number 
It specifies a TextArea's minimum width that can be displayed in the Form.
vMaxWidth String, Number 
It specifies a TextArea's maximum width that can be displayed in the Form.

If this value is smaller than vMinWidth, it will be replaced by the value of vMinWidth.
vMinHeight String, Number 
It specifies a TextArea's minimum height that can be displayed in the Form.
vMaxHeight String, Number 
It specifies a TextArea's maximum height that can be displayed in the Form.

If this value is smaller than vMinHeight, it will be replaced by the value of vMinHeight.
Sample Call:
var objComp = new TextArea();

objComp.init( "TextArea00", 30, 120, 196, 46 );
objComp.init( "TextArea00", 30, 120, 196, 46, null, null );
objComp.init( "TextArea00", null, null, 300, "400px", "80%", 300 );
objComp.init( "TextArea00", 0, 0, 200, 100, null, null, 300, 500, 200, 500 );
objComp.init( "TextArea00", "TextArea22:10", 300, null, null, "TextArea33:10", "20%", 300, 500, 200, 500 );
 
// Create Object  
var objTextArea = new TextArea();  
objTextArea.init("TextArea", 30, 120, 196, 46, null, null);
objTextArea.init("TextArea", null, null, 300, "400px", "80%", 300);
or
var objTextArea = new TextArea("TextArea", 30, 120, 196, 46, null, null);
var objTextArea = new TextArea("TextArea", null, null, 300, "400px", "80%", 300);
 
// Add Object to Parent Form  
this.addChild("TextArea", objTextArea); 
 
// Insert Object to Parent Form  
this.insertChild(1, "TextArea", objTextArea); 
 
// Remove Object form Parent Form  
this.removeChild("TextArea"); 
 
// Show Object  
objTextArea.show(); 
 
// Destroy Object  
objTextArea.destroy(); 
objTextArea = null;
