Description 
This property is used to define the direction of printing internal contents or components in a ListView.
Syntax 
ListView.rtl
Setting Syntax 
bRtl ::= undefined | 'true' | 'false'
 
var vRtl = this.ListView00.rtl;
undefined 
If undefined, the rtl property of the parent component will be applied.

This property will be undefined if you do not set this property or you delete the value of this property.
true 
Internal contents or components are printed from right to left.
The coordinate system applied to the ListView flows from right to left.
Text is printed from right to left in the ListView.
If there is a vertical scrollbar, it will be displayed on the left side.
false 
Internal contents or components are printed from left to right.
The coordinate system applied to the ListView flows from left to right.
Text is printed from left to right in the ListView.
If there is a vertical scrollbar, it will be displayed on the right side.
Remark 
- This property can be set only in Nexacro Studio and cannot be modified during execution of the application.

- This property is used to change the direction of printing contents. This property is usually used in regions like the Arab world where contents are laid out in a right-to-left flow direction on the screen.

- If all the rtl properties are undefined, beginning from a ListView all the way through to the Environment object, either of the below two cases will happen.
   > The rtl properties will be set based on the value of the locale property of the Environment.
   > If the locale property of the Environment is not set, the system's locale will be applied.
