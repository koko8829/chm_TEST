box-shadow
Supported Environments

[Mobile]
Nexacro Runtime Environment (NRE)

Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
The attribute to set the shadow effect on the MultiLineTextField.
Syntax 
MultiLineTextField.boxShadow[= strbshadow]
Setting Syntax 
strbshadow ::= 'none' | [<inset>] <offset> [<blur>'px'] [<spread>'px'] [<color>]

<offset> ::= <horizontal>'px' <vertical>'px'
<color> ::= <NamedColor> | <NumericColor>
 
* XCSS
box-shadow : 10px 10px 10px #888888;
box-shadow : inset 10px 10px 10px 5px #888888

* Script ( normal property )
this.MultiLineTextField00.boxShadow  = "10px 10px 10px #888888";
this.MultiLineTextField00.boxShadow  = "inset 10px 10px 10px 5px #888888";
<inset> 
The shadow effect is displayed inside the component.
It is operated as outset when omitted.
Not applicable in the NRE.
<offset> 
Set the position to display the shadow effect.

Set the position for displaying the shadow effect based on the top left location of the MultiLineTextField with a horizontal/vertical pixel value.
<horizontal> 
Set the horizontal position where the shadow effect is displayed.

When setting to a negative number, the shadow effect is displayed on the left side of the component.
It is entered in pixel units and the “px” unit cannot be omitted.
<vertical> 
Set the vertical position of the shadow effect to be displayed.

With negative number setting, the shadow effect is displayed on the top of the component.
It is entered in pixel units and the “px” unit cannot be omitted.
<blur> 
Set a blur on the shadow effect.
The blur is not applied when setting to a negative number.
It is entered in pixel units and the “px” unit cannot be omitted.
<spread> 
It expands or collapses the shadow.
It expands the shadow when set as a positive number, and collapses when set as a negative number.
Not applicable in the NRE.
<color> 
The color of the shadow effect can be set with the color name or color code.

It may operate differently depending on the browser or incorrectly if the value is not entered.
<NamedColor> 
Set a color name defined in advance such as “red” or “blue” as strings.
<NumericColor> 
This is the color code of the color to be displayed.

The color code can be set by a 6-digit hexadecimal value starting with the "#" symbol. (ex-"#FF00FF")
The color code can be set using the “rgb()” method that takes red, green, blue values as factors. (ex-"rgb(255,0,255)")
The color code can be set using the “rgba()” method that takes red, green, blue, alpha values as factors. (ex-"rgba(255,0,255,0.3)")
The color code can be set by the"hsl()" method that takes hue, saturation, lightness values as factors. (ex-"hsl(120,50%,50%)")
The color code can be set by the"hsal()" method that takes hue, saturation, lightness, alpha values as factors. (ex-"hsla(120,50%,50%,0.8)")
Remark 
- It is set as undefined and operated as "none" if the box-shadow attribute value is not set.

- "boxShadow" should be used for the attribute name when accessing with a script.

- Setting <blur> value too large may cause degradation as the requirement for system resources increases.

- <inset>, <spread> value does not support the NRE. 
  The designed changed during the attribute value setting can be checked from the design window of the nexacro studio.


◆ Restrictions in the Web Runtime Environment

- When setting the rgba(), hsl(), and hsla() methods in the attribute without using the XCSS, it may not be applied, depending on the browser version.
