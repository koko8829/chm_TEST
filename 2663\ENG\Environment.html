Description 
It saves and manages the environment information to be used in the project in the Environment object.
Remark 
- You can access the object by using the nexacro.getEnvironment() method in the script.
- When the attribute value of the Environment object (usewaitcursor, locale, etc.) by a script while executing the WRE, the changed attribute value of the Environment object is not reflected in the pop-up window that appears when executing the open method.

- It manages the cookies and variables to be used in the project.

- It manages the variance to be used commonly in the multiscreen environment.
