Property Type
Enum Expr Control Hidden ReadOnly Bind Collection StringResource
Description 
This property is used to set the type of a mouse pointer that appears when the mouse cursor is located in a TitleBarIconTextControl.
Syntax 
TitleBarIconTextControl.cursor
Setting Syntax 
cursor ::= 'none' | 'auto' | 'default' | 'copy' | 'crosshair' | 'help' | 'move' | 'not-allowed' | 'pointer' | 'progress' | 'text' | 'wait' | 'ew-resize' | 'e-resize' | 'w-resize' | 'ns-resize' | 'n-resize' | 's-resize' | 'nesw-resize' | 'ne-resize' | 'sw-resize' | 'nwse-resize' | 'nw-resize' | 'se-resize' 
* XCSS
cursor : default ;
"none" 
No mouse pointer will be shown.

* This value is not supported by the macOS NRE.
"auto" 
A pointer appropriate to the type of the relevant component

A text pointer will be displayed in Edit-affiliated components. 
Otherwise, an arrow pointer will be displayed.
"default" 
A default system pointer

Mostly an arrow pointer
"copy" 
An arrow with a small crosshair (+) beside

* This value is not supported by the macOS NRE.
"crosshair" 
A crosshair-shaped (+) pointer

* This value is not supported by the macOS NRE.
"help" 
An arrow with a small question mark beside

* In Safari, a pointer shaped as a question mark will appear.
* This value is not supported by the macOS NRE.
"move" 
A four-headed arrow 

* This value is not supported by the macOS NRE.
"not-allowed" 
A pointer shaped as a not-allowed symbol

* In Safari, an arrow with a not-allowed symbol beside will appear.
* This value is not supported by the macOS NRE.
"pointer" 
A hand-shaped pointer 

* This value is not supported by the macOS NRE.
"progress" 
This value presents a pointer varying according to OSes or environments.

There are examples like a spinning circle, arrow with a hourglass beside and arrow with a spinning circle beside.

* This value is not supported by the macOS NRE.
"text" 
An "I"-shaped pointer
"wait" 
This value presents a pointer varying according to OSes or environments.

There are examples like a spinning circle, hourglass and clock.
"ew-resize" 
A horizontally bidirectional arrow
"e-resize" 
A horizontally bidirectional arrow
"w-resize" 
A horizontally bidirectional arrow
"ns-resize" 
A vertically bidirectional arrow

* This value is not supported by the macOS NRE.
"n-resize" 
A vertically bidirectional arrow
"s-resize" 
A vertically bidirectional arrow
"nesw-resize" 
A diagonally bidirectional arrow of northeast and southwestwards
"ne-resize" 
A diagonally bidirectional arrow of northeast and southwestwards
"sw-resize" 
A diagonally bidirectional arrow of northeast and southwestwards
"nwse-resize" 
A diagonally bidirectional arrow of northwest and southeastwards

* This value is not supported by the macOS NRE.
"nw-resize" 
A diagonally bidirectional arrow of northwest and southeastwards

* This value is not supported by macOS.
"se-resize" 
A diagonally bidirectional arrow of northwest and southeastwards
Remark 
- If you do not set this property, the value will be undefined while this property will work as if the value is "auto".

- In the case where a mouse pointer enters a disabled sub-control of an enabled component, the mouse pointer will follow the "enable" status of the component rather than the "disable" status of the sub-control.
  In the case where both a component and its sub-control are disabled, but its parent is enabled, the mouse pointer will follow the "enable" status of the parent.
