strProps 
You call the addTarget() method to register a property and its values that are used to express an animation.

The third argument of the addTarget() method will become the value of the props property of the AnimationItem object.

You can define a value set by specifying a property and its values with a colon (:) between the two elements as a divider. You can define more than one value set.
In such a case where multiple properties are defined, commas are dividers separating each property.
<propName> 
This is a property that is used to create an animation.

You cannot define a read-only property.
<propValues> 
This is one or more values of the property defined to create an animation. The change in the values causes the animation effect.

If you want to specify multiple values, you should use brackets ([]) and commas to specify the values in the form of an array. The commas here function as dividers.
Remark 
- The animation effect is applied by changing the attribute value set in the props into the attribute value set in the props during the time set in the duration.

- The attribute value set in the props is applied evenly while the animation effect is executed.
   For example, when it is set as "left:[30,50,100]", the left value of the target component changes to 30 while executing the animation effect,
   and while the animation effect is executed for 50%, the left attribute value changes from 30 to 50, and then changes from 50 to 100 while finishing the execution to reach 100%.

- The animation effect would seem not working if only one attribute value is set to be changed in the
   props and the target component is the same as the current attribute value.
   For example, if the props attribute value is “left:100” and the left attribute value of the target component is 100, the attribute value changes from 100 to 100,
   hence there is no change although the animation effect is executed.

- The attribute where the animation effect is applied has the last value set in the props.
