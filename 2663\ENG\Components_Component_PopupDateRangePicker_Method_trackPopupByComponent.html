trackPopupByComponent
Supported Environments
[Desktop]
Nexacro Runtime Environment (NRE)
Windows macOS
Web Runtime Environment (WRE)
Edge Chrome Safari Firefox Opera
[Mobile]
Nexacro Runtime Environment (NRE)
Android iOS/iPadOS
Web Runtime Environment (WRE)
Android Default Web Browser iOS/iPadOS Default Web Browser
Description 
This is a method of displaying the PopupDateRangePicker component at a specific location based on the component delivered as an argument.
Syntax 
PopupDateRangePicker.trackPopupByComponent( strRangeposition, objComp, nXpos, nYpos [,nWidth ,nHeight [,strCallBackFunc [,bCapture]]] )
Parameters
Parameters Type Description
strRangeposition String 
Set a property that sets the starting position of the datepicker control, which displays the date to be indicated with priority on the screen.
- "start": Displays the month containing the date set in the startdate property value in the defaultrangeposition datepicker from the right side of the screen.
- "end": Displays the month containing the date set in the startdate property value in the defaultrangeposition datepicker from the left side of the screen.
objComp Object 
Set the component that will be the basis for coordinate calculation as an object.
nXpos Number 
Set the x coordinate position to display the PopupDateRangePicker component as a number based on the upper left corner of the objComp component.

The left of the PopupDateRangePicker component will be applied as the nXpos value.
When the PopupDateRangePicker component is only partially displayed on  the screen, the nXpos value will be changed automatically.
nYpos Number 
Set they coordinate position to display the PopupDateRangePicker component as a number based on the upper left corner of the objComp component.

The top of the PopupDateRangePicker component will be applied as the nYpos value.
When the PopupDateRangePicker component is only partially displayed on  the screen, the nYpos value will be changed automatically.
nWidth Number 
Set the width of the PopupDateRangePicker component as a number.

If the value is omitted, the value set during the design phase will be applied.
nHeight Number 
Set the height of the PopupDateRangePicker component as a number.

If the value is omitted, the value set during the design phase will be applied.
strCallBackFunc String 
This is the name of the callback function that will be called when the PopupDateRangePicker component is closed.

The component ID is passed as a parameter to the callback function.
The callback function must be defined in the form where the PopupDateRangePicker component is registered.
bCapture Boolean 
Set whether to give the Capture attribute to the PopupDateRangePicker component that pops up.

When set to true, a PopupDateRangePicker component with the Capture attribute will be displayed.
When set to false, the PopupDateRangePicker component will be displayed without the Capture attribute.

If the bCapture value is not set, it will be set to true.
Sample Call:
this.PopupDateRangePicker00.trackPopupByComponent( "start", this.Button00, 10, 10 );
this.PopupDateRangePicker00.trackPopupByComponent( "start", this.Button00, 10 ,10 ,200 ,200 ,"fn_Callback", false );
Return
None
Remark 
- If you call the trackPopup() method again while the PopupDateRangePicker component is displayed on the screen, an error will occur.
   Before executing the trackPopup() method, it is recommended to use the isPopup() method to check whether the PopupDateRangePicker component is displayed.

- If the trackPopup (or trackPopupByComponent) method of a PopupDateRangePicker component that has not yet been closed is called during the process of closing the PopupDateRangePicker component displayed on the screen, the method will not be processed.

- The trackPopup() method will display the PopupDateRangePicker component on the screen asynchronously (ASync), regardless of the async property value.
   In other words, the script will be executed after the execution of the trackPopup() method, irrespective of whether the PopupDateRangePicker component is displayed on the screen.

- If the PopupDateRangePicker component does not have a Capture attribute, when another component is selected, the PopupDateRangePicker component will disappear and the focus will move directly to the next component.
   If the PopupDateRangePicker component has a Capture attribute, when another component is selected, the PopupDateRangePicker component will disappear upon the first selection and the focus will move to the second selection.

- The return value passed from the closePopup() method will be passed to the callback function specified in strCallBackFunc.
